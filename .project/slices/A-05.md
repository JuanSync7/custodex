# Slice A-05/A-06 — `cdx coverage` CLI + `--fail-under` gate

Part of EPIC A. Surfaces the coverage system to the user and adds a gate flag.
Builds on A-01→A-04. (Combines roadmap A-05 + A-06.)

## Goal (validable)

`cdx coverage --config <cfg>` on a fixture repo prints the file-level and
public-symbol-level percentages plus the three baskets (documented count,
undocumented gaps listed, waived listed with reasons); `--json` emits the full
`CoverageReport` as JSON; `--fail-under N` exits **1** when
`percent_public_symbols < N` and **0** at/above; without `--fail-under` it always
exits 0 (informational, like `report`, not a gate like `check`). A bad config
prints `error: <msg>` to stderr and exits 1 with no traceback (K8). Proven by CLI
tests asserting output, JSON shape, and all exit codes.

## In scope — extend `custodex/cli.py` only

New command `coverage`:
```python
@app.command()
def coverage(
    config: Path = <--config default cdmon.yaml>,
    json_out: bool = <--json>,
    fail_under: float | None = <--fail-under, default None>,
) -> None: ...
```
Wiring (reuse existing helpers; config_dir = config.parent; root = config_dir/cfg.root):
1. `cfg = load_config(config)`
2. `inv = inventory.discover_files(root, include=cfg.coverage.include, exclude=cfg.coverage.exclude)`
3. `sym = inventory.discover_symbols(inv, root)`
4. `report = coverage.resolve_coverage(cfg, sym)`
5. human render OR `--json` (dump via pydantic `model_dump` / a `model_dump_json`).
6. if `fail_under is not None and report.percent_public_symbols < fail_under`: exit 1.
- Wrap the whole body in the per-command `try/except CodeDocMonitorError` →
  `error: <msg>` to stderr + `typer.Exit(1)` (the established K8 pattern — copy an
  existing command).
- Human output is deterministic (sorted baskets already). Show: two percentages,
  `documented N`, `undocumented N` then the gap list (`path::name (kind)`),
  `waived N` then list (`path::name — reason`). Keep it scannable.
- `--json` output must be valid JSON that round-trips the report (a test parses it
  back and checks a known field).

## CI wiring (the A-06 part) — `.gitlab-ci.yml`
Add an **informational, non-blocking** coverage step to `tests:offline` (or a
separate job) that runs `cdx coverage --config cdmon.yaml` and prints the
numbers. Do NOT make it a hard `--fail-under` gate yet — the engine's own new
modules (inventory/coverage/...) are not documented until EPIC H, so a hard gate
would fail today. Leave a comment: "enable `--fail-under` once H-02 raises
self-coverage." The gate *flag itself* is fully tested at the CLI level.

## Test plan (`tests/test_cli.py`, extend)
- `coverage` on a fixture config+repo: asserts the printed percentages and that a
  known gap symbol appears in the undocumented list and a waived one in the waived
  list with its reason.
- `--json`: parse stdout as JSON, assert `percent_public_symbols` and a basket
  entry are present and correct.
- `--fail-under`: above threshold → exit 0; below → exit 1; absent → exit 0 even
  with gaps present.
- bad config path/body → `error:` on stderr, exit 1, no `Traceback`.
- Use the pinned-Typer `CliRunner` pattern already in test_cli.py (monkeypatch.chdir
  for default-path cases — see the CDM-01 lesson).
- Coverage of new cli code ≥ 90%; whole-pkg gate ≥ 90%.
