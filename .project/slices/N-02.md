## Goal (validable)

`load_bundle`/`load_config_dir` now also enforce the REVERSE invariant: every
`*.yaml` unit file physically present in `config/cdmon/` (excluding reserved
stems `index`, `ignore`, `doc-style`) MUST be listed in `index.yaml`'s `units`;
an on-disk unit missing from the index is a loud `ConfigError`. A new
`cdmon index` command regenerates `config/cdmon/index.yaml`'s `units:` list from
the on-disk units (idempotent, K7) while preserving the globals + frontmatter.
Proven by: a fixture with an unindexed unit raises; `cdmon index` on a drifted
index rewrites it to list all units and re-running is a no-op.
