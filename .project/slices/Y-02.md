# Slice Y-02 — config-sync engine + HTTP endpoints (git / local)

Part of EPIC Y. Builds on Y-01 (persistence). Adds the server-side engine that
reads a repo's `config/cdmon/`, merges to `MonitorConfig`, computes drift +
coverage, persists documents/code-refs/sync-run, and exposes it over HTTP. See
`.project/spec/CONFIGV2.md` §4.

## Goal (validable)

`POST /repos/{repo_id}/sync {mode: "git"|"local"}` runs a real sync against the
registered repo's `local_path`: **git** mode reads `config/cdmon/` (and source)
at the repo's default branch (`main`); **local** mode reads the working tree. It
upserts `config_documents`/`config_code_refs` (Y-01), records a `sync_runs` row,
and returns the run summary `{fully_synced, mode, document_count, code_ref_count,
commits_ahead, drift}`. `GET /repos/{id}/documents?sync_kind=` returns documents
with nested code-refs (the relationship view); `GET /repos/{id}/sync-state` returns
the latest run. Proven by an end-to-end test that creates a REAL temp git repo
with a `config/cdmon/` + source + docs, registers it with `local_path`, and:
git-sync at main populates the DB + reports fully_synced when clean; a working-
tree code edit on a feature branch → local-sync reports `commits_ahead>=1` and
non-empty drift; the documents endpoint returns the right doc→code_refs tree.

## In scope

### Model additions (additive, K6)
- `RepoIdentity` (sinks.py) + `RegistrationPayload` (registry.py): add optional
  `local_path: str | None = None` and `default_branch: str | None = None`. These
  round-trip through the existing `/repos` register route + the repos table JSON
  (no migration — payload is stored as full JSON). Confirm existing tests pass.

### Engine — new module `custodex/configsync.py` (pure-ish; git is the
only side-effect, read-only on the repo)
```python
class SyncResult(BaseModel):           # frozen
    documents: tuple[ConfigDocument, ...]
    code_refs: tuple[ConfigCodeRef, ...]
    run: SyncRun

def read_config_at(local_path: Path, *, mode: str, branch: str,
                   now: str) -> tuple[ConfigBundle, Path, GitInfo]: ...
def run_sync(local_path: Path, repo_id: str, *, mode: str,
             default_branch: str = "main", now: str) -> SyncResult: ...
```
- **local mode:** operate directly on `local_path` working tree. `load_bundle`
  there; `detect()` for drift; coverage via `resolve_coverage` over the derived
  `effective_coverage`. `head_commit` = `git rev-parse HEAD`; `branch` =
  `git rev-parse --abbrev-ref HEAD`; `main_commit` = `git rev-parse <default>`;
  `commits_ahead` = `git rev-list --count <default>..HEAD` (0 if on default or
  no git). `fully_synced` (local) = `commits_ahead == 0 AND no drift`.
- **git mode:** materialize the repo at `<default_branch>` READ-ONLY without
  disturbing the working tree — use `git worktree add --detach <tmp> <branch>`,
  run the same load+detect+coverage there, then `git worktree remove --force`.
  (If `git worktree` is unavailable, fall back to `git archive <branch> | tar -x`
  into a tmp dir — pick one, make it robust + cleaned up in a finally.)
  `fully_synced` (git) = `no drift at main` (the central baseline).
- Build `ConfigDocument`/`ConfigCodeRef` from the merged config + bundle
  (`unit_for_document` for unit attribution; `sync_kind` = mode; `ref` = the
  resolved commit; `synced_at = now`). Build `SyncRun` with counts, git info,
  `fully_synced`, and a `drift` dict summarizing drift kinds + coverage percent.
- Inject `now` (K10). Subprocess git via the repo's existing subprocess pattern
  (look at how `pr.py`/`syncpr.py`/`open-docs-pr` shell out to git — reuse that
  helper for consistency + testability). Loud K8 on git failure / missing
  `local_path` / unknown mode.

### Endpoints — `server/app.py`
- `POST /repos/{repo_id}/sync` body `{mode}` — bearer-auth like other writes
  (reuse `_verify_token`). 404 if repo unknown; 400 if `mode` invalid or repo has
  no `local_path`. Calls `run_sync(...)`, then `store.replace_config(...)` +
  `store.add_sync_run(...)`, returns the run summary JSON.
- `GET /repos/{repo_id}/documents?sync_kind=` — returns
  `[{document fields..., code_refs: [...]}]` by joining `config_documents_for` +
  `code_refs_for` (stable order). Used by the W-01 relationship view.
- `GET /repos/{repo_id}/sync-state?sync_kind=` — `latest_sync_run` (or null).
- Inject the server clock via the existing app clock seam (find how app.py
  timestamps records, e.g. detected_at) so tests are deterministic.

## Test plan

- `tests/test_configsync.py` (new): build a REAL temp git repo (git init, config
  user, commit) with config/cdmon + source + in-sync docs. Test `run_sync`
  local + git modes: clean → fully_synced True, correct counts; edit a symbol in
  the working tree → local drift non-empty + git mode (main) still clean;
  feature-branch commit → commits_ahead>=1. Assert ConfigDocument/CodeRef
  attribution (unit, sync_kind, ref). Cleanup leaves no leftover worktrees.
- `tests/test_server_sync.py` (new) or extend parity suite: via FastAPI
  TestClient over BOTH stores — register a repo with local_path, POST /sync
  (git+local), assert response summary + DB rows; GET /documents returns the
  doc→code_refs tree; GET /sync-state returns the latest run; auth enforced
  (401/403 like other writes); 400 on bad mode / missing local_path; 404 unknown
  repo.
- Determinism: inject `now`; no wall-clock leakage. ≥90% branch coverage of new
  code; full suite green.

## Constraints
K1 (sync READS the repo; never writes the user's tree — git worktree is a tmp
checkout, removed after), K6 (additive models), K8 (loud), K10 (injected clock;
deterministic ordering). Offline: real local git only, no network.
