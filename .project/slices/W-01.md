## Goal (validable)

A new Documents page at `/repos/:id/documents` lists every config document for a
repo (doc_id, audience, region_keys, unit) and lets the user expand a document to
reveal its `code_refs` (path + symbols) ‚Äî the relationship view. Data comes from
`GET /repos/{id}/documents?sync_kind=git`. Proven by Vitest tests: loading/error/
empty states, the list renders the document fields, expanding a row reveals its
code_refs, and the API client calls the right URL with the sync_kind param.

## API contract (captured, authoritative)

`GET /api/repos/{id}/documents?sync_kind=git` ‚Üí array of:
```json
{ "document": {"repo_id","doc_id","path","audience","unit","region_keys":[...],
               "sync_kind","ref","synced_at"},
  "code_refs": [ {"repo_id","doc_id","path","symbols":[...],"unit","sync_kind"} ] }
```
ALWAYS pass `sync_kind` (default `git`) ‚Äî the unfiltered endpoint mixes git+local
rows. (Add a small sync_kind toggle git|local if cheap; default git.)
