# Slice H-02 — self-dogfood: document the engine's own new modules

EPIC H. The headline self-improvement: code-doc-monitor documents ITSELF. The
EPIC A–G modules (inventory, coverage, syncpr, pr, registry, similar, promotion,
doctor, server/*) are undocumented — self-coverage is **12.2%** public symbols
(135/1106). This slice raises it substantially and LOCKS it with a hard CI gate.

## Goal (validable)
`cdmon coverage --config cdmon.yaml` public-symbol coverage rises from 12.2% to a
committed target (aim **≥ 80%**; document what's documented vs deliberately
**waived** so the remainder is accounted for — lossless); every new doc is
in-sync (`cdmon check` + `cdmon lint` exit 0); the CI coverage step flips from
informational to a HARD `cdmon coverage --fail-under <T>` gate at the achieved
level. Proven by: the coverage number, a clean check/lint, and the CI gate config.

## In scope
### Document the undocumented engine modules (mechanical heal does the tables)
- Add `DocumentSpec`s to `cdmon.yaml` for the new modules. Group sensibly into a
  few eng-guide docs (don't make 15 tiny files) — e.g.:
  - `docs/api/coverage-system.md` ← inventory.py + coverage.py
  - `docs/api/pr-loop.md` ← syncpr.py + pr.py
  - `docs/api/central-client.md` ← registry.py (+ the sinks envelope already in remediation)
  - `docs/api/learning.md` ← similar.py + promotion.py
  - `docs/api/server.md` ← server/app.py + server/store.py + server/db.py (eng-guide)
  - `docs/api/ops.md` ← doctor.py (+ cli.py public surface if useful)
  (Adjust groupings as sensible; reuse `cdmon new-doc <id>` to scaffold each so it
  passes lint, then `cdmon monitor --apply` to fill the symbol tables.)
- Add each new doc to the `api-index` landing page (it indexes all docs).
- For genuinely-not-worth-documenting symbols (private-ish public helpers, the
  `if __name__` mains, generated `__all__`), use the `coverage.waive` list WITH a
  reason — so the remaining gap is explicit and the % reflects real intent.

### Lock the gate
- `.gitlab-ci.yml`: change the informational coverage step to
  `cdmon coverage --config cdmon.yaml --fail-under <T>` where `<T>` is a couple
  points below the achieved number (headroom). Comment the target.

## Test plan
- `tests/test_dogfood.py` (extend): assert the new docs are in sync (the existing
  dogfood in-sync assertion should now cover them) and that
  `cdmon coverage` public-symbol % ≥ the committed target (a test reads the
  coverage report and asserts the threshold — so coverage can't silently regress).
- All new docs pass `cdmon lint` (structure) — add to the lint dogfood check.
- The FULL Python gate stays green (the heals don't break tests). ≥90% code coverage
  unchanged (this is DOC coverage, not test coverage — different metric).

## Process
- This is mostly mechanical: scaffold doc → add spec → `monitor --apply` fills
  generated regions → `check`/`lint` clean. Write only minimal human prose (the
  `>` purpose line + any intro). Do NOT hand-edit generated `symbols` regions.
- Re-run `cdmon coverage` after each batch to track the climb; stop when ≥ target
  or when the remainder is justifiably waived.
- It may be large — if context grows, batch by doc and keep going (the slice is
  done when the threshold + gate + green suite hold).

## Out of scope
Telemetry (H-01), regression corpus (H-03), gap-issue surfacing (H-04). Changing
engine CODE (this is docs + config + CI only; if you find a genuine bug while
documenting, note it in `.project/problems/`, don't fix it here).

## Constraints
K2 (generated tables from code), K7 (idempotent heal), K10, K9 (suite green). The
coverage threshold test makes the self-improvement DURABLE (can't regress silently).
Update STATUS/LESSON/ROADMAP; this is the core "the monitor improves itself" proof.
