# Slice TDOC-02 — Demo encapsulates the test→test-doc mirror 1:1

EPIC TDOC. TDOC-01 proved the mirror + dogfooded it on the smoke boundary. This
slice makes the **demo** (`demo/`, the adopter showcase) demonstrate the mirror
FULLY and 1:1: every demo test file under `demo/tests/` gets exactly one test-doc
under `demo/test-docs/`, exactly as every demo source file maps to a doc.

## Why
The demo is the canonical 1:1 showcase (and `cdx trace` proves feature↔demo↔test
completeness). To "fully encapsulate" the new feature it must monitor its OWN tests
the same way it monitors its source — a clean, complete test→test-doc map.

## Goal (validable)
1. **1:1 mapping.** `demo/config/cdmon/tests.yaml` (`dir-covered: [tests]`) declares
   one `eng-guide` test-doc per demo test file (`test_engine`, `test_io`,
   `test_model`, `test_scheduler` → `demo/test-docs/test_*.md`), each with a
   `symbols` region over the test file's functions. `demo/tests/__init__.py` waived.
2. **In sync + clean.** From `demo/`: `cdx check`, `cdx lint` clean;
   `cdx coverage` reflects the new test universe (tests unit 100%); the source
   `scheduler.py` gap is preserved (the walkthrough's link→100% still reaches 100%).
3. **rpt regenerated.** `cdx rpt --write` updates `demo/config/cdmon/coverage.rpt`
   to the new overall % (was 80% with only source; now higher with tests at 100%).
4. **Docs + count pins updated.** `demo/README.md` (config table, doc table, the
   coverage prose) and `demo/DEMOS.md` (a new DEMO entry + every stale count) match
   reality. The demo e2e count/doc-id pins (`test_demo_e2e.py`,
   `test_gitrepo_sync_e2e.py`) include the new test-docs.
5. **Console.** `scripts/seed_demo.py` keeps showing every demo doc on both the
   git (Documents) and local (Mapping) partitions — the test-docs flow through the
   same projection automatically.

## Design
- **`demo/config/cdmon/tests.yaml`** (new unit, mirrors TDOC-01's dogfood unit).
- **`demo/test-docs/test_*.md`** (new, generated via `cdx new-doc` + human
  blockquote).
- **`demo/config/cdmon/index.yaml`**: waive `tests/__init__.py`; `cdx index`.
- **`demo/config/cdmon/coverage.rpt`**: regenerated by `cdx rpt --write`.
- **Count-pin ripple** (the #1 breakage source — grep `demo` doc-id sets + the old
  coverage % across `tests/` and `demo/*.md` and update EVERY site).

## Test plan
- **system (`tests/system/test_demo_e2e.py`):** `_DEMO_DOC_IDS` + `_DEMO_CODE_REFS`,
  `document_count`/`code_ref_count` updated to include the four test-docs.
- **system (`tests/system/test_gitrepo_sync_e2e.py`):** the demo `doc_ids` cases +
  the store-parity / demo-as-git doc-id set assertions include the new ids.
- Demo gate from `demo/`: `cdx check`/`lint`/`coverage`/`index` clean; full
  offline pytest green; `cdx trace` still 1:1 (feature added in TDOC-03).

## Out of scope
The feature-catalog entry + `cdx trace`/`wiki` (TDOC-03); the frontend sections
(TDOC-04, already delivered). No engine code.
