# Slice P-02 — Tiered fingerprint structure + which-tier-moved reporting

EPIC P. Replaces the opaque `surface_hash` string with a structured
`SurfaceFingerprint` (per-tier digests + a `composite`), reports WHICH tier moved
on a `HASH` drift, and lands the real `ReviewRecord.schema_version` → `1.1.0`
minor bump. Built on P-01's additive-key discipline: the `composite` is
byte-identical to today's `surface_hash()`, so no stored fingerprint is
invalidated and there is no re-baseline.

## Goal (validable)
1. **Structured fingerprint, identity unchanged.** `DocumentSurface.fingerprint(*,
   include_body)` returns a frozen `SurfaceFingerprint(signature, docstring, body,
   composite)`. `composite == surface_hash(...)` for every audience/flag (asserted
   against the frozen P-01 golden literals). `surface_hash` now delegates to
   `fingerprint().composite`.
2. **Which-tier-moved.** On a `HASH` drift, `Drift.drifted_tiers` names the tier(s)
   whose digest moved — proven by three tests: a body-only change → `("body",)`, a
   signature change → `("signature",)`, a docstring change → `("docstring",)`. An
   OLD doc with only a composite fingerprint (no stored tiers) → `()` with the
   composite-only fallback message.
3. **Schema bump (K6).** `ReviewRecord.schema_version == "1.1.0"`;
   `ReviewRecord.drifted_tiers` is additive (default `()`, appended last) — a
   legacy `"1.0.0"` JSONL line still `model_validate_json`s with `drifted_tiers ==
   ()`. The emitted JSON Schema carries the new property.

## Design
- **extract.py.**
  - `SurfaceFingerprint(BaseModel)` (frozen, `extra="forbid"`): `signature: str`,
    `docstring: str | None`, `body: str | None`, `composite: str`, plus
    `drifted_against(other) -> tuple[str, ...]` (sorted tier names whose digest
    differs; a present-vs-`None` digest counts as a move).
  - `DocumentSurface.fingerprint(*, include_body=False)`: builds the SAME composite
    payload as the pre-P2 hash (so `composite` bytes are unchanged) AND three
    diagnostic per-tier digests via a shared `_hash_payload`. The **signature** tier
    hashes the structural surface (signature-only symbols + `records`) so a
    records-only change is attributed to it; **docstring** is `None` unless eng-guide;
    **body** is `None` unless the opt-in body tier is on and a function/method
    contributes a body digest. `surface_hash` → `fingerprint(...).composite`.
- **manifest.py.** Additive front-matter accessors: `stored_fingerprint_tiers(doc)
  -> SurfaceFingerprint | None` (None on an old doc) and `set_fingerprint_tiers(meta,
  fp)` writing `cdm.fingerprint_tiers` (omitting `None` sub-tiers; composite stays in
  `cdm.fingerprint`, the identity). Imports `SurfaceFingerprint` from `extract` (no
  cycle — `extract` imports neither).
- **One-shared-truth stamping.** `heal.regenerate_regions`/`_corrected` and
  `layout.scaffold_doc` compute `fp = surface.fingerprint(include_body=...)` ONCE
  and stamp BOTH `set_fingerprint(meta, fp.composite)` and `set_fingerprint_tiers(meta,
  fp)` — detect and stamp can never disagree on the tiers.
- **drift.py.** `Drift.drifted_tiers: tuple[str, ...] = ()`. The `HASH` branch
  computes `current_fp = surface.fingerprint(...)`, compares `current_fp.composite`
  to the stored composite (identity, unchanged), and when they differ derives
  `drifted_tiers` from `current_fp.drifted_against(stored_fingerprint_tiers(doc))`
  (or `()` if no stored tiers) plus a tier-naming detail message.
- **schema.py / monitor.py.** `ReviewRecord.schema_version = "1.1.0"`;
  `ReviewRecord.drifted_tiers: tuple[str, ...] = ()` appended last; `monitor`
  threads `drifted_tiers=drift.drifted_tiers` into the record.

## Schema / migration (K6)
This IS the P2 bump P-01 deferred. `surface_hash` (the composite) is unchanged, so
**no stored `cdm.fingerprint` is invalidated**; turning the body tier on remains the
only deliberate re-baseline (P-01). The new `cdm.fingerprint_tiers` block is additive
front matter — an old doc without it is detected exactly as before (composite-only
message). `ReviewRecord` minor-bumps to `1.1.0` with one additive field; old `1.0.0`
records still parse. ARCHITECTURE.md pins `SurfaceFingerprint`, `fingerprint()`, the
manifest accessors, `Drift.drifted_tiers`, and the bumped `ReviewRecord`.

## Test plan
- **unit (test_extract.py):** composite == surface_hash (both audiences, both flags);
  composite == frozen golden; signature tier always present; docstring tier eng-only;
  body tier gated (None off / present eng-on / None user-on, K3); `drifted_against`
  per tier + identical → `()`; signature tier folds records; frozen.
- **unit (test_manifest.py):** tiers round-trip; absent → None (old doc); None
  sub-tiers round-trip; `set_fingerprint_tiers` additive (preserves composite +
  region hashes).
- **integration (test_drift.py):** HASH drift reports body/signature/docstring tier;
  old doc (composite only) → `()` + fallback message.
- **integration (test_schema.py):** default `1.1.0`; `drifted_tiers` additive +
  round-trips; legacy `1.0.0` line still parses with `drifted_tiers == ()`; schema
  includes the property.
- **system/e2e (test_monitor.py, test_system.py):** a record carries
  `drifted_tiers == ("body",)` for a body-only change (flag on); heal stamps the
  per-tier digests into front matter and a body change reports tiers `("body",)` end
  to end, with the human-readable detail naming the tier.

## Dogfood
Adding the public symbols (`SurfaceFingerprint`, `fingerprint`,
`set_fingerprint_tiers`, `stored_fingerprint_tiers`) drifts the eng-guide API docs on
the signature tier; rehealed via `cdmon monitor --apply --config config/cdmon`
(the reheal also stamps `cdm.fingerprint_tiers` into the dogfood docs). `cdmon check`
+ `cdmon lint` exit 0.

## Out of scope
Non-Python extractors (P-03); region anchors (P-04). P-02 keeps the body tier opt-in
(P-01's flag) and does NOT change the composite bytes.

## Constraints
K0 (no target knowledge, stdlib only), K1 (detect pure), K3 (user-guide composite
bytes constant; docstring/body tiers excluded for user-guide), K6 (additive minor
bump; old records + old docs still valid), K7 (idempotent reheal), K10 (deterministic
sorted, clock-free per-tier digests).
