# Slice N-06 — repo-root unification + `index --check` fix + dir-layout regression test

Remediation slice (from Phase-1 validation). Fixes two confirmed bugs that make
the `config/cdmon/` dir layout non-functional for `cdx check`/`monitor`/coverage,
and adds the regression test whose absence hid them. See `.project/spec/CONFIGV2.md`.

## Defects being fixed

1. **HIGH — repo-root divergence.** `Monitor.root` (`monitor.py:~101`) and
   `drift.detect()` (`drift.py:~111`) compute the repo root as
   `config_dir / config.root`, while `config._resolve_repo_root`
   (`config.py:~1261`, used by `effective_coverage`, doc-style `templates_root`,
   and `cdx rpt`) computes `config_dir.parent / root`. For the dir layout
   (`config_dir = <repo>/config/cdmon`, default `root = ".."`) these diverge by
   one level: Monitor resolves to `<repo>/config` (WRONG); the coverage/rpt path
   resolves to `<repo>` (RIGHT). For single-file (`config_dir = <repo>`,
   `root = "."`), `_resolve_repo_root` would resolve to the PARENT of the repo
   (WRONG) — it only happens not to be called on that path today. No single
   `root` value satisfies both formulas.

2. **MEDIUM — `cdx index --check` never reports in-sync.** `regenerate_index`
   refreshes the frontmatter `updated:` via the wall-clock `_now()` seam, and
   `--check` compares the FULL text, so the timestamp always differs → always
   exit 1 in real use (the passing test only works because it monkeypatches the
   clock on both sides).

## Goal (validable)

- ONE repo-root resolver, used by every consumer, correct for BOTH layouts.
  `cdx check`, `cdx monitor --apply`, and `cdx coverage` run correctly
  against an on-disk `config/cdmon/` repo (code_refs + docs resolve under the
  true repo root; coverage scans the right files). Proven by a NEW regression
  test that builds an on-disk dir-layout repo with real code + docs and asserts
  `detect()` finds zero drift when in sync, finds drift after a code edit, and
  `monitor --apply` heals it — the exact scenario the per-slice tests skipped.
- `cdx index --check` returns exit 0 when only the `updated:` timestamp would
  change (i.e. units are in sync), and exit 1 only on a real units change.
  Proven WITHOUT monkeypatching the clock (use two genuinely different wall times
  or assert the comparison ignores `updated:`).

## In scope

### Defect 1 — unify repo-root resolution
- Add ONE public helper in `config.py`:
  `resolve_repo_root(config_dir: Path, root: str) -> Path` defined as
  `Path(os.path.normpath(Path(config_dir) / root))`. (Convention: `root` is
  relative to `config_dir`, matching single-file semantics and Monitor's existing
  formula.) Export it.
- DECISION (locked): for the dir layout `root` is relative to `config/cdmon`, so
  the repo-root-from-`config/cdmon` default becomes `"../.."` (NOT `".."`).
  Change the `IndexFile.root` default to `"../.."`. Update CONFIGV2.md §1.2 +
  ARCHITECTURE.md to state the convention plainly: "`root` is the repo root
  relative to the directory the config lives in (`config/cdmon/` for the dir
  layout, so default `../..`; the repo root for a single file, so default `.`)."
- Reimplement `_resolve_repo_root` to delegate to `resolve_repo_root` (drop the
  `.parent`), so `effective_coverage`, doc-style `templates_root`, and `cdx rpt`
  now use `config_dir / root`.
- Point `Monitor.root` and `drift.detect()` at `resolve_repo_root(config_dir,
  config.root)` (replacing ad-hoc `config_dir / config.root`) so all five
  consumers share ONE formula. Verify single-file behavior is unchanged
  (`config_dir = repo`, `root = "."` → repo).
- Update every dir-layout test fixture that set `root: ".."` to `root: "../.."`
  (grep tests/ for the v2 fixtures from N-01..N-05). Also update any docs/spec
  examples. The full suite must stay green after the convention change.

### Defect 2 — `index --check`
- In `cdx index --check` (cli.py), compare the regenerated index to the on-disk
  one IGNORING the frontmatter `updated:` line (e.g. normalize both by blanking
  the `updated:` value before diffing), so a pure timestamp delta is NOT drift.
  A real units-list change still exits 1 with a diff. Keep the rewrite path
  (non-`--check`) refreshing `updated:` as before.
- Update/replace `test_cli_index_check_in_sync` so it proves in-sync→exit 0
  WITHOUT relying on a frozen clock (e.g. write the index, then run `--check`
  with a DIFFERENT injected time and assert exit 0).

### Regression test (the gap)
- New `tests/test_dirlayout_e2e.py`: build an on-disk repo under tmp_path with
  `config/cdmon/` (index `root: "../.."`, 1–2 units, ignore.yaml, doc-style +
  templates), real `.py` source, and matching docs with managed regions. Assert,
  invoking through the dir-layout resolution path (CLI `_resolve_config` or
  `load_config_dir` + `detect`/`Monitor`):
  - clean repo → `detect()` returns no drift; `cdx check` exit 0;
  - edit a code symbol → `detect()` reports drift; `cdx check` exit 1;
  - `cdx monitor --apply` heals it; re-check exit 0 (K7 idempotent);
  - `cdx coverage` over the dir layout reports the correct % (code under the
    true repo root, not `config/`).
