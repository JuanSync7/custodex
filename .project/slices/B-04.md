# Slice B-04/B-05 — mixed-authorship e2e + lint of modes

Final EPIC-B slices. Proves the implemented authority modes coexist in ONE doc,
and teaches `cdmon lint` about modes. (Combines roadmap B-04 + B-05.)

> **Scope decision:** the three *fully-implemented* modes are exercised
> end-to-end: `generated` (mechanically rewritten), `human` (preserved + advisory),
> `llm-seeded` (fill-then-lock). **Pure-`llm` prose authoring** — a no-renderer
> region whose body an LLM writes from scratch and re-authors on drift — is
> DEFERRED to a dedicated slice (call it B-06) near the backend/EPIC-D remediation
> work, because it needs real backend prose generation, not just a mode flag.
> B-04 must DOCUMENT this: a region declared `llm` today behaves like `llm-seeded`
> pre-lock (engine renders if a renderer exists) OR like `human` if no renderer —
> pick the safe behavior, assert it, and note B-06 will replace it.

## Goal (validable)
(B-04) A single fixture doc carrying four regions — one `generated`, one `human`,
one `llm-seeded`, one declared `llm` — survives a code change + `monitor --apply`
with each region behaving per its mode: generated regenerated, human byte-identical
+ advisory, llm-seeded fill-then-lock honored, llm per the documented interim
rule. Proven by a system test asserting all four region outcomes in one run.
(B-05) `cdmon lint` reports each managed region's declared mode and (for
llm-seeded/human) its lock/advisory state, and flags a structural problem the
layout standard defines (e.g. a `region_modes` entry is already validated at
config-load, so lint surfaces the *state*, not re-validation). Proven by a lint
test on a fixture with mixed modes.

## In scope
### B-04 — `tests/test_system.py` (+ any tiny glue)
- Build a real fixture doc with 4 managed regions + `region_modes` for three of
  them (generated is the default). Drive `regenerate_regions`/`monitor.run` and
  assert the four outcomes. Reuse the B-02/B-03 machinery (`preserve`,
  `locked_region_ids`, `region_is_locked`) — ideally NO new engine code beyond
  deciding+encoding the interim `llm` rule (document it in code + LAYOUT_STANDARD).
- If the interim `llm` rule needs a one-line branch in drift/heal, keep it minimal
  and additive; otherwise treat `llm == llm-seeded` for now (simplest) and say so.

### B-05 — `code_doc_monitor/layout.py` + `cdmon lint`
- Extend `lint_doc`/`lint_config` (read the current layout.py API first) to report,
  per managed region, `mode` and lock/advisory state, using the public manifest
  helpers (`stored_region_hash`, `region_is_locked`) + `spec.mode_for`. This is a
  STATE surface, not a new gate — keep `lint`'s existing pass/fail semantics; add
  an informational line or a `--modes` view (your call; document it).
- Update `docs/LAYOUT_STANDARD.md` §7 (region authority) to document
  `cdm.region_hashes` and the interim `llm` behavior.
