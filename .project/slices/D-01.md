# Slice D-01/D-02 — resolution outcome edge + `cdmon resolve`

First EPIC-D slices. Turns the write-only review log into a learning substrate by
capturing the **human outcome** of a handled drift. An outcome is a SEPARATE
append-only event linked to a `ReviewRecord` by `record_id` (the review log stays
append-only/immutable, K5 — we never mutate a record).

## Goal (validable)
`cdmon resolve <record_id> --resolution accepted [--by NAME] [--text ...]` appends
a `ResolutionRecord` to an append-only resolutions log; `read`/`summarize` join
records↔resolutions so a reviewer sees which drifts were accepted/overridden/
rejected and which still need a human; the resolutions log is versioned + additive
(an unknown future field doesn't break old readers, K6); resolving an unknown
`record_id` is a loud error (K8). Proven by: append+read round-trip, the join
summary, a back-compat parse test, and the unknown-id error.

## In scope
### `schema.py` — `ResolutionRecord` (new, public, versioned, K6)
```python
class Resolution(str, Enum):
    ACCEPTED = "accepted"        # the proposed FIX was right, merged as-is
    OVERRIDDEN = "overridden"    # human kept the fix idea but rewrote it (resolved_text)
    REJECTED = "rejected"        # the fix was wrong; drift stands / handled elsewhere
    INVALIDATED = "invalidated"  # a human judged the drift a non-event

class ResolutionRecord(BaseModel):       # frozen, extra="forbid"
    schema_version: str = "1.0.0"
    record_id: str                       # FK → ReviewRecord.record_id
    resolution: Resolution
    resolved_text: str | None = None     # the human's final text when OVERRIDDEN
    resolved_by: str | None = None
    resolved_at: str                     # ISO, injected (K10)
    note: str | None = None
```
`resolution_record_schema()` = `model_json_schema()` (one source of truth, K6).

### `reviewlog.py` (extend) or `resolutionlog.py` (new) — append-only JSONL
- `append_resolution(path, rec)`, `read_resolutions(path) -> list[ResolutionRecord]`
  (mirror the existing `append`/`read_all` exactly: append mode, parent dirs,
  blank-line skip, corrupt line → loud `SchemaError` with line no., K8).
- A join helper: `resolved_index(resolutions) -> dict[record_id, ResolutionRecord]`
  (last-write-wins if a record is resolved twice — document the choice) and an
  extended `summarize` (or a new `summarize_with_resolutions(records, resolutions)`)
  reporting counts of resolved-vs-unresolved and by-resolution.

### `cli.py` — `cdmon resolve`
- `cdmon resolve <record_id> --resolution R [--by NAME] [--text TEXT] [--note N]
  [--config] [--log PATH]`: validates the `record_id` EXISTS in the review log
  (loud K8 if not), appends a `ResolutionRecord` (timestamp injected via the same
  `now` seam monitor uses), prints a confirmation. Default resolutions log path
  alongside the review log (e.g. `.cdmon/resolutions.jsonl`).
- Optionally extend `cdmon report` to show resolved/unresolved counts (additive).

## Test plan
- `ResolutionRecord` round-trips; `resolution_record_schema()` valid; enum values.
- append/read resolutions JSONL round-trip; corrupt line → SchemaError(line no.).
- back-compat (K6): a resolutions line WITHOUT a future field still parses; default
  fields tolerated.
- join/summary: 3 records, 2 resolved (1 accepted, 1 overridden) → summary shows
  2 resolved / 1 unresolved + by-resolution counts.
- CLI: resolve a real record_id → appends + confirms; unknown id → clean K8 error
  (no traceback); `--text` stored for OVERRIDDEN.
- determinism: injected `now` → stable `resolved_at`.
- Additive: ALL prior tests green. ≥90% new code; gate ≥90%.

## Out of scope
Retrieval/similarity (D-03), exemplars (D-04), promotion (D-05/06), the central
sink emitting resolutions (EPIC E). `cdmon resolve` writes LOCAL JSONL only.

## Constraints
K5 (review log immutable; outcome is a separate append-only event), K6
(versioned + additive schema), K8 (loud on unknown id / corrupt line), K10
(injected timestamp), K9. This is the substrate D-03..D-06 mine.
