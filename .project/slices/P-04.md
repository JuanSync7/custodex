# Slice P-04 — Anchors (region ⇄ stable symbol identity)

EPIC P (final slice). Binds a managed region to the **stable identities** of the
symbols it documents — an `anchor_id` derived from the qualified name, never the
line number — recorded additively in front matter. On a `HASH` drift, detect now
classifies the change by symbol identity: an empty anchor delta means the SAME
symbols changed internally (a move/reorder or a body/docstring edit — a *re-bind*,
not a structural change), while a nonempty delta means a symbol was
added/removed/renamed. Combined with P-02's per-tier digests this realizes the
goal's distinction: **"symbol moved (re-bind, no structural drift)" vs "body
changed (real drift)".**

## Goal (validable)
1. **Stable identity.** `anchor_id(qualified_name) -> str` (sha256[:16] of the
   NAME only) + `Symbol.anchor_id` — invariant under a pure code move (different
   lineno, same name → same anchor), changed by a rename. Proven directly.
2. **Recorded per region.** Heal + scaffold stamp `cdm.region_anchors[id]` (sorted
   anchor_ids) for the symbol-table region (`blocks.REGION_KEYS`), additively
   (composite fingerprint, tiers, region hashes preserved). Idempotent.
3. **Classified drift.** On a `HASH` drift, `Drift.anchors_added`/`anchors_removed`
   report the symbol-identity delta vs the stamped set: a body-only change →
   BOTH empty + `drifted_tiers == ("body",)` (re-bind, internal change); a public
   symbol added → its anchor in `anchors_added`; a symbol removed → its anchor in
   `anchors_removed`. A pre-P4 doc (no stored anchors) → both empty (fallback).

## Design
- **extract.py.** `anchor_id(name)` (module fn) + `Symbol.anchor_id` property
  delegating to it. Lineno-free and deterministic (K10); a rename is a different
  identity (the anchor is NOT a rename-tracker — a renamed symbol reads as
  removed+added, which is the honest signal).
- **manifest.py.** Additive `cdm.region_anchors` accessors —
  `stored_region_anchors(doc, region_id) -> tuple[str, ...] | None` (None pre-P4)
  and `set_region_anchors(meta, region_id, anchors)` (stores SORTED; mirrors
  `set_region_hash`'s additive, copy-forward discipline so the set survives later
  heals).
- **heal.py / layout.py.** When the engine authors a region whose id is in
  `REGION_KEYS` (the symbol-table region), stamp
  `set_region_anchors(meta, id, tuple(s.anchor_id for s in surface.symbols))` —
  one-shared-truth with the fingerprint/tier stamping. Stamped independent of
  B-03 mode tracking (anchors describe identity, not authorship).
- **drift.py.** `Drift.anchors_added`/`anchors_removed` (additive, default `()`).
  On a `HASH` drift, compute the current surface's anchor set and diff it against
  the stamped anchors of the doc's symbol-table region(s); a nonempty delta is
  appended to the human detail (`+N/-M`). Empty stored anchors (pre-P4) ⇒ empty
  delta (back-compat).

## Schema / migration (K6)
No `ReviewRecord`/surface wire change and no composite re-baseline. `Drift` gains
two additive fields (default `()`); `cdm.region_anchors` is additive front matter
(an old doc lacking it classifies with an empty delta). ARCHITECTURE.md pins
`anchor_id`, `Symbol.anchor_id`, the manifest accessors, and the `Drift` fields.

## Test plan
- **unit (test_extract.py):** anchor_id deterministic/short; changes on rename;
  `Symbol.anchor_id` matches the function; survives a code move (different lineno,
  same anchor).
- **unit (test_manifest.py):** region_anchors round-trip (stored sorted); absent →
  None (pre-P4); additive (preserves fingerprint/tiers/region_hashes).
- **integration (test_drift.py):** body-only change → anchors stable + tiers
  `("body",)`; added public symbol → `anchors_added`; removed symbol →
  `anchors_removed`; pre-P4 doc → empty delta.
- **system/e2e (test_system.py):** heal stamps `region_anchors`; a body change
  keeps anchors stable while dropping a public symbol reports it removed — through
  the monitor/CLI path.

## Dogfood
The new public surface in `extract.py` (`anchor_id`, `Symbol.anchor_id`) drifts the
eng-guide API docs (signature tier); rehealed via `cdx monitor --apply` (the
reheal also stamps `cdm.region_anchors` into the dogfood docs). `cdx check` +
`cdx lint` exit 0.

## Out of scope
Re-binding a `lines`-selected ref by identity after a move (would couple
extraction to stored front matter); rename TRACKING (a rename reads as
removed+added — the honest, lossless signal); surfacing anchors on the
`ReviewRecord` (the drift fields suffice for the goal).

## Constraints
K0 (identity from the name, no target knowledge), K1 (detect pure), K6 (additive
fields + front-matter key; no wire/composite change), K7 (idempotent anchor
stamping), K10 (deterministic, lineno-free anchor; sorted storage).
