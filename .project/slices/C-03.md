# Slice C-03 — bot-PR opener (injected GitLab transport)

EPIC C. Turns a `SyncResult` (C-02) into a **docs merge request**: a branch + a
commit of the healed doc files + an MR. Network is INJECTED (K4) — tests use a
fake transport and assert the payload; the real GitLab HTTP calls are the only
uncovered leaf (mirror the `sinks.py`/`backends.py` inject-the-leaf discipline).

## Goal (validable)
Given a non-empty `SyncResult` over a fixture repo, `plan_docs_pr` builds a
deterministic `MergeRequestPlan` (branch name derived from a hash of the patch →
stable + unique-per-change; title `docs: sync …`; description listing
`changed_paths`; the healed file contents as commit actions); `open_docs_pr` with
a FAKE transport submits exactly that plan and returns its result; `--dry-run`
returns the plan WITHOUT calling the transport; an EMPTY `SyncResult` is a no-op
(no branch, no MR, returns None). Proven by fake-transport tests asserting the
plan fields, the dry-run no-call, and the empty no-op. NO network in any test.

## In scope — `code_doc_monitor/pr.py` (new) + `cli.py`
```python
class MergeRequestPlan(BaseModel):     # frozen, extra="forbid"
    source_branch: str                 # e.g. "cdmon/docs-sync-<patch-hash[:12]>"
    target_branch: str                 # default "main"
    title: str                         # "docs: sync to <ref>" (ref optional)
    description: str                    # changed_paths + a note this is bot-generated
    files: tuple[tuple[str, str], ...]  # (repo-relative path, NEW content) — commit actions
    labels: tuple[str, ...] = ()

class PRTransport(Protocol):
    def submit(self, plan: MergeRequestPlan) -> dict: ...   # returns provider response (e.g. {"web_url": ...})

class GitLabTransport:                 # DEFAULT — stdlib urllib only (K0), no `python-gitlab` dep
    # project id + token from env (e.g. CI_PROJECT_ID / a token env var, injected at construction);
    # submit(): create branch from target, create a commit with file actions, open the MR — each via
    # a tiny urllib leaf that is the ONLY `# pragma: no cover` line (K4).

def plan_docs_pr(sync: SyncResult, root: Path, *, target_branch="main",
                 ref: str | None = None, branch_prefix="cdmon/docs-sync",
                 labels=()) -> MergeRequestPlan | None: ...   # None if sync.patch == ""
def open_docs_pr(sync, root, *, transport: PRTransport, dry_run=False, **plan_kw) -> dict | None: ...
```
- `plan_docs_pr` reads each `sync.changed_paths` file's CURRENT content from `root`
  (these are the healed docs — caller runs `sync-pr` apply first) → `files`.
  Branch name = `f"{branch_prefix}-{sha256(sync.patch)[:12]}"` (deterministic, K10).
- `open_docs_pr`: empty sync → return None (no-op). `dry_run` → return the plan as a
  dict (no transport call). Else `transport.submit(plan)`.
- `cli.py`: `cdmon open-docs-pr [--config] [--dry-run] [--target BRANCH] [--ref REF]`
  — runs `sync_pr(apply=True)` then `open_docs_pr`; `--dry-run` prints the plan and
  mutates nothing meaningful for the MR (sync still heals locally unless you also
  pass a dry sync — keep it simple: in `--dry-run` use `sync_pr(dry_run=True)` so
  NOTHING is written and the plan is computed from the would-be-healed content).
  Default GitLab transport built from env; loud K8 error if required env is missing.

## Test plan (`tests/test_pr.py` + `tests/test_cli.py`)
- non-empty sync → `plan_docs_pr` returns a plan: assert branch (deterministic hash),
  title, description contains each changed path, `files` carry the healed contents.
- `open_docs_pr` with a FAKE transport (records the plan) → returns the fake's
  response; the captured plan equals the planned one.
- `--dry-run` / `dry_run=True` → transport NOT called (fake asserts zero calls).
- empty sync → `open_docs_pr` returns None, transport not called.
- determinism: same patch → same branch name.
- default `GitLabTransport`: cover construction + the "missing env → loud error"
  path; the real urllib calls are the only `# pragma: no cover` leaf (stub it to
  cover the build-default branch, exactly like sinks/backends).
- Additive: all prior tests green. ≥90% new code; gate ≥90%.
