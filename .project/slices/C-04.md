# Slice C-04/C-05 ‚Äî loop-safety + provenance

Final EPIC-C slices. (C-04) Structurally prevents the docs-PR from re-triggering
the heal loop. (C-05) Stamps where a heal came from. Closes the PR-driven loop
safely.

## Goal (validable)
(C-04) `should_sync(changed_files, config)` returns **False** when EVERY changed
file is a managed doc path (a bot doc-only commit ‚Üí don't re-heal) and **True**
when any non-doc file changed; a `cdmon should-sync` CLI exits 0 (proceed) / 1
(skip) accordingly; the `.gitlab-ci.yml` heal job uses it so a doc-only commit
runs no heal. (C-05) `ReviewRecord` carries an additive `source_sha: str | None`
(old logs still parse, K6); `monitor` stamps it from a passed ref/`--ref`/env; the
MR description (C-03) already shows it. Proven by: `should_sync` truth-table tests
(doc-only‚Üískip, mixed‚Üírun, empty‚Üískip), CLI exit-code tests, a schema round-trip
proving old records without `source_sha` still load, and a monitor test asserting
the stamped sha.

## In scope
### C-04 ‚Äî loop-safety
- `syncpr.py` (or a small helper): `should_sync(changed_files: Iterable[str],
  config: MonitorConfig) -> bool`. Managed doc paths = `{d.path for d in
  config.documents}` (POSIX-normalized). Return `True` iff at least one changed
  file is NOT a managed doc path. Empty changed set ‚Üí `False` (nothing to do).
  Pure, deterministic.
- `cli.py`: `cdmon should-sync [--config] [FILES...]` (or read newline-separated
  from stdin) ‚Üí exit 0 if `should_sync` is True, 1 if False. K8 clean errors.
- `.gitlab-ci.yml`: the heal/sync job runs `git diff --name-only` for the push and
  pipes to `cdmon should-sync` (or a guard step) so a doc-only (bot) commit skips
  heal ‚Äî breaking the loop structurally. Keep valid YAML; comment WHY.

### C-05 ‚Äî provenance
- `schema.py`: add `source_sha: str | None = None` to `ReviewRecord` (ADDITIVE,
  after existing fields, default None ‚Äî K6; `cdmon schema` re-emits it). A pre-C-05
  JSONL line (no `source_sha`) must still `model_validate_json` (test it).
- `monitor.py`: `Monitor`/`run` accept an optional `source_sha` (or read a passed
  value) and stamp every `ReviewRecord` with it; default None keeps today's
  records valid. Thread from `cli.py` via `--ref/--source-sha` (and let it fall
  back to `$CI_COMMIT_SHA` if you add env support ‚Äî document the precedence).
- The `pr.open-docs-pr` description already carries `ref`; ensure the same value
  can flow to `source_sha` (document the one-source-of-truth: CLI passes ref to
  both monitor and the PR plan).

## Test plan
- `should_sync`: doc-only set ‚Üí False; one `.py` among docs ‚Üí True; empty ‚Üí False;

- CLI `should-sync`: exit 0 vs 1 for the above; bad config ‚Üí clean error.

- `.gitlab-ci.yml` still `yaml.safe_load`s.
