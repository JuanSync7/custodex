# Slice H-01/H-04 — telemetry (underperformers) + surface gaps as a dry-run issue

EPIC H. Two "surface what needs improving" features: (H-01) server-side telemetry
that flags which drift shapes the backend handles poorly (high escalate / high
human-override), and (H-04) a client command that turns coverage gaps into an
issue payload for the monitored repo's tracker (injected transport, dry-run).

## Goal (validable)
(H-01) `GET /repos/{repo_id}/telemetry` returns, per `(drift_kind, audience)`
shape: counts, escalation_rate, override_rate (fraction of resolutions that were
`overridden`), and promotion candidates (reuse `detect_promotions` server-side) —
sorted worst-first; proven by TestClient tests with seeded records+resolutions
asserting exact rates + ordering. (H-04) `cdx surface-gaps [--config]
[--dry-run]` builds a deterministic issue payload from the coverage gaps
(undocumented public symbols, grouped by suggested owner from A-07), submits via an
INJECTED `IssueTransport` (real GitLab/GitHub issues API the only uncovered leaf);
`--dry-run` prints the payload without calling; no gaps → no-op. Proven by
fake-transport tests + dry-run-no-call. NO network.

## In scope
### H-01 — server telemetry (`server/app.py` + store/db read)
- `RepoTelemetry`/`ShapeStat` view models (computed, like RepoStatus — NOT the
  shared schema): per `(drift_kind, audience)` → `{count, escalations,
  escalation_rate, overrides, override_rate}`, plus a `promotion_candidates`
  list (`detect_promotions` over that repo's records+resolutions).
- `GET /repos/{repo_id}/telemetry` → `RepoTelemetry{repo_id, shapes: [...] sorted
  by escalation_rate desc then override_rate desc, candidates: [...]}`. Read-only,
  open (like the other GETs). Add any needed store aggregation (or compute from
  `records_for`+`resolutions_for`) on BOTH InMemoryStore and SqlStore.
- TestClient tests: seed a repo with known escalate/override mix → assert exact
  rates, worst-first ordering, and the promotion candidates.

### H-04 — `cdx surface-gaps` (client, injected issue transport)
- `custodex/issues.py` (new): `IssuePlan{title, body, labels}`,
  `IssueTransport` Protocol (`submit(plan)->dict`), `GitLabIssueTransport`/
  `GitHubIssueTransport` (stdlib urllib; real POST the only `# pragma: no cover`),
  `plan_coverage_issue(report, suggestions) -> IssuePlan | None` (None if no gaps;
  deterministic body listing undocumented public symbols grouped by
  `suggest_owners`), `open_coverage_issue(..., transport, dry_run)`.
- `cli.py`: `cdx surface-gaps [--config] [--dry-run] [--provider gitlab|github]`
  — runs discover→coverage→suggest, builds the plan, `--dry-run` prints it; else
  submits via the provider transport (url/token from env, loud K8 if missing).
- Tests: gaps → plan with the right title/body (a known undocumented symbol
  appears, grouped under its suggested owner); fake transport submit; dry-run
  no-call; no-gaps → no-op None; default-transport build + missing-env loud path
  (real POST the only uncovered leaf).
