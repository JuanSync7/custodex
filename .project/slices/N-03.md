# Slice N-03 — `ignore.yaml` + `.gitignore` merge + `source-files-format` coverage scoping

Part of EPIC N (CONFIG-V2). Builds on N-01/N-02. See `.project/spec/CONFIGV2.md`
§1.1 (source-files-format rule) and §1.3 (ignore.yaml).

## Goal (validable)

When loaded via the dir layout, coverage is scoped by the units and the ignore
set with NO change to the coverage engine: `load_config_dir` derives the merged
`MonitorConfig.coverage.include` from each unit's `dir-covered` × its
`source-files-format`, and `coverage.exclude` from `ignore.yaml` (its `patterns`
plus, when `gitignore: true`, the repo `.gitignore` translated to globs). The
result: a file under a unit's `dir-covered` whose extension is NOT in that unit's
`source-files-format` (e.g. a `.log`/`.rpt`) is NOT in the coverage universe (it
never shows as "uncovered"); an ignored file is likewise excluded. Proven by an
automated test: a fixture repo with `agent/backend.py`, `agent/run.log`,
`agent/build.rpt`, `agent/__pycache__/x.pyc` under a unit `dir-covered:
[agent], source-files-format: ['.py']` with `ignore.yaml` patterns — and the
resulting `CoverageReport` universe contains ONLY `agent/backend.py`.

## In scope

`custodex/config.py`:
- Add `IgnoreFrontmatter` (cdmon-config-version=="2.0.0", source, updated) and
  `IgnoreFile` model: `gitignore: bool = False`, `patterns: tuple[str,...] = ()`.
- `load_ignore_file(path) -> IgnoreFile` (frontmatter+body; loud K8).
- `gitignore_to_globs(text: str) -> tuple[str, ...]`: translate `.gitignore`
  lines to the inventory `**`-glob semantics. Handle: skip blanks + `#` comments;
  skip negations (`!...`) — emit nothing for them and (optionally) note; trailing
  `/` → directory anywhere → `**/<name>/**`; leading `/` → root-anchored →
  `<rest>` and `<rest>/**`; embedded `/` (path) → as-is plus `<p>/**`; bare token
  (no slash) → `**/<tok>` and `**/<tok>/**`; keep `*`/`?` wildcards. Deterministic
  (sorted, deduped output). FIRST read `custodex/inventory.py` `_translate`
  to confirm exact `**` semantics (does `a/**/b` match `a/b`? does a leading
  `**/` match zero segments?) and write the translation + tests to MATCH it.
- `effective_coverage(bundle: ConfigBundle, repo_root: Path) -> CoverageConfig`:
  - `include`: for each unit, for each `d` in `dir-covered`, for each `ext` in
    `source-files-format`: the glob(s) needed so files DIRECTLY in `d` AND in
    nested subdirs match (verify both `d/*ext` and `d/**/*ext` forms against
    inventory `_translate`; include whichever forms are required — add a test
    proving a file directly under `d` and one nested both match). Sorted/deduped.
  - `exclude`: `ignore.patterns` ∪ (`gitignore_to_globs(repo_root/.gitignore)` if
    `ignore.gitignore` and the file exists) ∪ the existing default excludes
    (`**/.*/**`, `**/__pycache__/**`, `**/.venv/**`) so dot-dirs/caches stay out.
  - `waive`: from `index.coverage.waive` (unchanged).
- Wire into `load_config_dir`: the returned `MonitorConfig.coverage` IS
  `effective_coverage(...)`. (Keep `load_bundle` returning the bundle; have
  `load_config_dir`/`load_bundle` resolve the repo_root as `config_dir.parent`
  joined with `index.root`.) Existing single-file `load_config` is unchanged.
- Expose `effective_coverage` + `gitignore_to_globs` in `__all__`.

Do NOT modify `coverage.py`/`inventory.py` — they already consume
`config.coverage.include/exclude/waive`. The whole point is the derived coverage
flows through the existing engine untouched.

## Test plan (`tests/test_config_ignore.py`, new)

- **unit (gitignore_to_globs)**: comments/blanks skipped; `__pycache__/` →
  `**/__pycache__/**`; `*.log` → `**/*.log`; `/dist` → `dist`+`dist/**`;
  `docs/**/*.html` preserved; `!keep.py` produces nothing; determinism.
- **unit (effective_coverage)**: include built from dir-covered × format; a
  nested file and a direct file both covered; exclude = patterns ∪ gitignore ∪
  defaults; waive carried from index.
- **integration (the goal test)**: real fixture repo + config/cdmon; run the
  actual coverage path (`resolve_coverage` over `discover_files`/`discover_symbols`
  using the derived include/exclude) and assert the universe is exactly the
  format-matched, non-ignored files — `.log`/`.rpt`/`.pyc` absent; assert the
  coverage percent denominator excludes them.
- **gitignore on/off**: with `gitignore: false`, a `.gitignore`-only pattern does
  NOT exclude; with `true`, it does.
- ≥90% branch coverage of new code; full suite green.
