# Slice T-01 — Drift Ticket core (the agent fills a structured ticket)

## Why
Today a handled drift surfaces as a **one-liner** (`ProposedFix.rationale` /
`ReviewRecord.cause`). A human cannot meaningfully *validate* a one-liner. Replace
it (additively) with a **DriftTicket** — a Jira-style structured artifact the
backend (agent) fills and a human reviewer validates: title, summary, severity,
affected symbols, root cause, proposed change + diff, and an explicit acceptance
checklist. The existing `ResolutionRecord` stays the human validation EVENT.

## In scope
### New module `custodex/ticket.py` (pure — K1/K10, no new deps — K0)
- `class TicketSeverity(str, Enum)`: `LOW="low"`, `MEDIUM="medium"`, `HIGH="high"`.
- `class TicketStatus(str, Enum)`: `PROPOSED="proposed"`, `VALIDATED="validated"`,
  `CHANGES_REQUESTED="changes-requested"`, `REJECTED="rejected"`.
- `class AcceptanceCheck(BaseModel)` (frozen, extra="forbid"): `text: str`,
  `auto_satisfied: bool = False` (the agent's claim that the proposed change
  already satisfies this criterion; the human still confirms).
- `class DriftTicket(BaseModel)` (frozen, extra="forbid"):
  `schema_version: str = "1.0.0"`, `ticket_id: str`, `title: str`, `summary: str`,
  `severity: TicketSeverity`, `drift_kind: str`, `doc_id: str`, `doc_path: str`,
  `region_id: str | None = None`, `audience: str`,
  `affected_symbols: tuple[str, ...] = ()`, `root_cause: str`,
  `proposed_change: str`, `change_kind: str` (`"region"|"whole-doc"|"none"`),
  `diff: str = ""`, `acceptance_criteria: tuple[AcceptanceCheck, ...] = ()`,
  `verdict: str`, `recommended_action: str`.
- `def build_ticket(drift: Drift, result: BackendResult, surface: DocumentSurface,
  *, ticket_id: str) -> DriftTicket` — PURE, DETERMINISTIC. Derives every field
  from inputs already in `monitor._record_for` (no clock, no I/O):
  - `title`: `f"[{severity.value.upper()}] {drift.kind.value} in {drift.doc_id}"` +
    region suffix `f" · region {drift.region_id}"` when present. Keep < ~100 chars.
  - `summary`: a 1–2 sentence plain-language statement built from `drift.detail`
    and `result.cause` (e.g. `f"{drift.detail} The remediation agent's read: {result.cause}"`).
  - `severity` heuristic (deterministic):
    `HIGH` if `verdict == ESCALATE` or `drift.kind == UNHEALABLE` or `not drift.healable`;
    `LOW` if `verdict == INVALIDATE`; else `MEDIUM`.
  - `affected_symbols`: sorted public symbol names from `surface.symbols`
    (`s.name for s in surface.symbols if s.is_public`), as a tuple.
  - `root_cause`: `result.cause`.
  - `change_kind`: `"region"` if `result.fix` has a `region_id`/`new_region_body`;
    `"whole-doc"` if it has `new_doc_text`; `"none"` if `result.fix is None`.
  - `proposed_change`: from `result.fix.rationale` when a fix exists; else a
    sensible no-fix line (INVALIDATE: "No documentation change needed — …";
    ESCALATE: "Needs a human author — …"). Use `result.cause` for context.
  - `diff`: `drift.diff` (already a unified diff for REGION drifts; "" otherwise).
  - `recommended_action`: `"Apply the proposed fix"` (FIX) / `"Dismiss as
    not-applicable"` (INVALIDATE) / `"Escalate to a human author"` (ESCALATE).
  - `acceptance_criteria` (deterministic, verdict-aware list of `AcceptanceCheck`):
    - FIX: "Proposed text matches the current code surface" (auto_satisfied=True),
      "No human-owned region is modified" (auto_satisfied=True),
      "Document still passes `cdx lint`" (auto_satisfied=False).
    - INVALIDATE: "Change is confined to non-public surface (docstring/comment/private)"
      (auto_satisfied=True), "Public API is unchanged" (auto_satisfied=False).
    - ESCALATE: "A human has authored the missing/owned content" (auto_satisfied=False),
      "The new content is in sync with the code surface" (auto_satisfied=False).
- `def ticket_status(resolution: ResolutionRecord | None) -> TicketStatus` — pure
  mapping: `None → PROPOSED`, `accepted → VALIDATED`, `overridden → CHANGES_REQUESTED`,
  `rejected/invalidated → REJECTED`. (Used by the server/dashboard later; defined here.)
- `__all__` lists all public names. Module docstring follows the repo style
  (terse, cites K-invariants). Keep it dependency-free (stdlib + pydantic only).

### `custodex/schema.py` (ADDITIVE — K6)
- Add to `ReviewRecord` a new field, appended **LAST** (after `source_sha`):
  `ticket: DriftTicket | None = None`. Default None ⇒ every pre-T-01 JSONL line
  still `model_validate_json`s (write a back-compat test). Import `DriftTicket`
  from `.ticket`. NOTE: `ticket.py` imports `Drift`/`BackendResult`/`DocumentSurface`
  only inside `build_ticket`'s module scope — make sure there is **no import cycle**
  (`schema.py` must import `ticket.DriftTicket`; `ticket.py` imports from `drift`,
  `backends`?, `extract`). `backends.py` imports `schema` → importing `backends`
  from `ticket` would cycle. AVOID: in `build_ticket` type the `result` param via
  `from .backends import BackendResult` — this WILL cycle (schema←ticket←backends←schema).
  RESOLUTION: do **not** import `BackendResult` in `ticket.py`. Type `result` as a
  small structural Protocol (`class _ResultLike(Protocol): verdict: Verdict; cause: str;
  fix: ProposedFix | None`) OR accept the already-decomposed primitives. Prefer:
  `build_ticket(*, drift, verdict, cause, fix, surface, ticket_id)` taking
  `verdict: Verdict`, `cause: str`, `fix: ProposedFix | None` — all from `schema`,
  which `ticket.py` may import (schema does not import ticket-the-builder, only the
  model; keep the model import in schema and the builder usage in monitor). Verify
  `python -c "import custodex"` has no circular import.

### `custodex/monitor.py`
- In `_record_for`, build the ticket and pass it to the `ReviewRecord(...)`:
  `ticket=build_ticket(drift=drift, verdict=result.verdict, cause=result.cause,
  fix=result.fix, surface=surface, ticket_id=<id>)`. Derive `ticket_id` deterministically
  — reuse the record id: `ticket_id=f"CDM-{record_id}"` (compute `record_id` first).
  The `_target_record` placeholder may leave `ticket=None` (it is never persisted).
