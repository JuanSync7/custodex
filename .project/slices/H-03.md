# Slice H-03 — regression corpus from lessons & known limitations

Final EPIC-H slice (and the program's capstone). Turns the hard-won lessons and
captured limitations into a runnable **regression corpus** so each learned failure
mode stays fixed forever — the monitor's self-improvement becomes durable.

## Goal (validable)
A `tests/regression/` corpus encodes, as executable tests, the standing invariants
and the specific failure modes captured across the build (LESSON_LEARNT.md +
`.project/problems/*`). Each corpus case names the lesson it guards and FAILS if
the regression returns. Proven by: the corpus runs green in the default suite, and
a documented "remove the fix → corpus reds" check for at least the top invariants.

## In scope — `tests/regression/`
Curate the highest-value, still-true guards. Read `.project/LESSON_LEARNT.md` and
every `.project/problems/*.md`, and encode the durable ones, e.g.:
- **Self-coverage invariant** (H-02): `cdmon coverage --config cdmon.yaml`
  public-symbol % stays ≥ the committed threshold AND every engine module is owned
  — plus a fixture proving an UNLISTED `code_doc_monitor/**` module is detected as
  a gap (the H-01/H-04 finding).
- **Human-region never auto-edited** (B-02): a `human` region survives `monitor
  --apply` byte-identical; advisory persists across a fingerprint heal until edited
  (B-03 fix).
- **llm-seeded fill-then-lock** (B-03): the three-phase property.
- **Heal idempotence / no trailing newline** (CDM-03): regenerate→clean→regenerate
  False.
- **Audience invalidation** (CDM-07): comment-only edit drifts eng, not user.
- **Loop-safety** (C-04): doc-only commit → `should_sync` False.
- **Schema additivity / back-compat** (D-01, C-05): a pre-field JSONL record still
  parses.
- **Reporting never raises** (E-01): a down transport queues, doesn't throw.
- **Rule resolves with zero backend calls** (D-06).
- **The dogfood stays in sync** (the existing assertion, referenced here).
Prefer REFERENCING existing tests where they already pin a lesson (import/parametrize
or a thin re-assertion) over duplicating — the corpus is a curated INDEX of
invariants, not a copy. Where a lesson has no test yet, ADD one.

- `tests/regression/README.md` (or a module docstring): each case → the lesson id
  it guards (`[B-02]`, `[CDM-07]`, …) so a failure points straight to the writeup.
- A `regression` pytest marker (optional) so the corpus can be run alone
  (`pytest -m regression`) AND is included in the default suite.

## Test plan
- The corpus is itself the test. It must pass in the full suite.
- For 2–3 top invariants, include a comment documenting the "break-it" check the
  author ran (e.g. "temporarily set healable=True for human → this reds") — don't
  commit the break, just document that the guard bites.
- Additive: all prior tests green; no engine code change (this is tests +
  optionally a marker registration in pyproject). ≥90% maintained.

## Process
- Pure test curation. If you find a lesson that is NO LONGER TRUE (superseded),
  note it (don't encode a stale guard). If you find a real GAP (a lesson with no
  guarding test), add the test — that's the point.
- Run the FULL gate green. If you register a `regression` marker, update
  `pyproject` addopts/markers and confirm `pytest -m regression` selects the corpus
  and the default run still includes it.

## Dogfood
Tests only → no tracked-module change; confirm `cdmon check` + `cdmon coverage
--fail-under 95` exit 0. Update STATUS/LESSON/ROADMAP; **mark EPIC H + the whole
EPIC-2 program COMPLETE** in STATUS/ROADMAP with a closing summary.

## Out of scope
New features. Re-testing everything (curate the DURABLE invariants, not every unit).

## Constraints
K9 (suite green; corpus is additive), K10 (deterministic). The corpus is the
program's durable memory: a learned failure that has a corpus guard cannot
silently come back.
