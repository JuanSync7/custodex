# Slice B-06 тАФ pure-`llm` prose authoring (replace the interim `llm`==`generated` rule)

The one deferred EPIC-B slice. Today a region declared `mode: llm` with **no
mechanical renderer** surfaces as `UNHEALABLE`/ESCALATE forever (the safe interim
rule documented in ARCHITECTURE.md ┬з"B-04 тАФ interim `llm` rule"). B-06 makes such a
region **backend-authored prose**: the backend writes its body from the code surface
and **re-authors it when the surface drifts**, so the doc stays fresh without a human
and without a mechanical renderer. The offline `MockBackend` authors a deterministic
stand-in so the default suite stays offline (K4) and idempotent (K7/K10).

## Goal (validable)
A fixture doc with a `mode: llm` region that has **no renderer** (a prose section,
not a symbol table):
1. when the code surface it documents changes, `cdmon monitor --apply` (default
   `MockBackend`, offline) **re-authors that region's prose** from the surface
   instead of escalating it `UNHEALABLE`;
2. a second `monitor --apply` is a **no-op** (idempotent тАФ same surface тЗТ identical
   authored body тЗТ `cdmon check` CLEAN);
3. the never-silently-rot guarantee holds: if there is genuinely **no authoring
   path** (a purely mechanical `regenerate_regions` heal with no backend), the
   region still surfaces (loud), never silently stale;
4. a **`human`** region in the same doc is still **byte-identical** after the heal
   (B-02 guarantee unbroken тАФ `llm` authoring must not touch human-owned bodies).
Proven by a system/e2e test driving (1)-(4) end-to-end, plus unit tests on the new
drift classification and the MockBackend prose rule.

## Design (bake these in тАФ they were derived from the seam)
- **`backends.FixRequest`** gains `region_mode: RegionMode = RegionMode.GENERATED`
  (ADDITIVE, K6 тАФ every existing FixRequest/test unchanged). `monitor.run` sets it
  from `spec.mode_for(drift.region_id)` when building the request.
- **`drift.detect`** тАФ in the `region_id not in known` branch (drift.py ~L167): a
  region whose `mode is RegionMode.LLM` and has no renderer is **prose-authored**,
  NOT unknown. Replace the unconditional `UNHEALABLE` for that case with:
  - code moved (`stored != current` fingerprint) тЗТ a `REGION` drift, `healable=True`
    (the backend authors it), detail e.g. `"llm-authored region {id!r} is stale;
    backend will re-author from the current surface"`, carrying the audience + a
    `_short_diff` if useful.
  - code unchanged тЗТ **not drift** (the prose stands; an `llm` body legitimately
    differs from any render).
  - A region that is `not in known` and **not** `llm` mode keeps today's
    `UNHEALABLE` behavior (genuinely no renderer + no authoring path).
  - A `llm` region WITH a renderer keeps today's behavior (it's `in known` тЖТ
    mechanically rendered + kept in sync). Unchanged.
- **`backends.MockBackend.propose`** тАФ add a rule BETWEEN current rule 1 (renderer
  region тЖТ FIX) and rule 2: if `drift.kind is REGION`, `req.region_mode is
  RegionMode.LLM`, and `expected_region(...)` is `None` (no renderer), author a
  **deterministic, idempotent** prose body from `req.surface` (e.g. a stable
  sentence enumerating the public symbols the section covers, audience-aware) and
  return `FIX` with `new_region_body=<prose>`. Same surface тЗТ identical body (K10).
  Keep it simple and documented; it is the offline stand-in for "what an LLM would
  write", exactly as rule 3 is for whole-doc.
- **`backends.build_prompt`** тАФ when `drift.kind is REGION` and `req.region_mode is
  RegionMode.LLM`, append a short clause: "This region is LLM-authored PROSE (it has
  no mechanical renderer): write a clear, audience-appropriate prose description of
  the surface below; do NOT emit a symbol table." (Real backends only; the mock and
  the existing single-shot prompt tests stay green тАФ additive wording, assert it's
  present for an llm-region request.)
- **`heal.apply_fix` / `_corrected`** тАФ a `llm` region is NOT human and NOT locked,
  so it is authorable (injected) тАФ confirm it is never added to the preserve/locked
  set. CRITICAL idempotence point: the whole-doc HASH fix (`render_corrected`) must
  **leave a no-renderer `llm` region's body intact** (preserve it, never blank it) тАФ
  the separate REGION fix authors the new prose. Verify `_corrected` already
  preserves no-renderer regions; if it blanks/drops them, fix that (preserve current
  body for any region with no renderer, regardless of mode) and add a test.

## Test plan
- `test_drift`: a `llm` no-renderer region, code moved тЗТ REGION healable=True (not
  UNHEALABLE); code unchanged тЗТ no drift; a non-`llm` no-renderer region тЗТ still
  UNHEALABLE.
- `test_backends`: MockBackend authors deterministic prose for an llm REGION drift
  (assert exact body for a known surface), and the SAME body on a second call
  (idempotent). build_prompt includes the prose clause for an llm region request and
  NOT for a generated one.
- `test_heal`: a HASH whole-doc fix preserves the llm region body byte-identical; an
  llm REGION fix injects the new prose; a `human` region alongside is untouched.
- `test_system`/e2e: the four-goal scenario above (re-author тЖТ idempotent no-op тЖТ
  human untouched тЖТ no-backend mechanical heal still surfaces it). Extend or mirror
  `test_mixed_authorship_four_regions_e2e` rather than duplicating it.
- Additive: ALL prior tests stay green (the interim-rule assertion in
  `test_mixed_authorship_four_regions_e2e` that expects UNHEALABLE for the `llm`
  region MUST be updated to the new authored behavior тАФ do it, and note the change).
  тЙе90% coverage on new code; whole-package gate тЙе90%.
