# Slice R-01 — Feature catalog schema + loader (golden-reference foundation)

EPIC R, phase R1. The first slice of the "golden feature reference + traceability
+ wiki" epic. Before we can map demos/tests/source to features, features must be
**named, typed, and loadable** from a single machine-readable source of truth.
R-01 builds that foundation: a `featurecatalog.py` with a frozen pydantic `Feature`
model and a `load_catalog()` that aggregates `feature-doc/catalog/*.yaml` into a
validated, deterministic `FeatureCatalog` — loud (K8) on every malformed input.

R-01 ships the engine + a tiny seed catalog fixture; the FULL catalog population
(every subsystem) is R-02+. No demos/tests/source mapping yet (R2–R6 read this).

## Goal (validable)
1. **Typed feature.** `Feature` is `frozen=True, extra="forbid"`; `id` matches
   `FEATURE_ID_RE` (`FEAT-<SUBSYS>-<NNN>`); `modules` is non-empty; `status` ∈
   {implemented, planned, deprecated}. A bad id / empty modules / extra key →
   `CatalogError` (K8).
2. **Aggregating loader.** `load_catalog(dir)` reads every `*.yaml` under `dir`
   (sorted), each `{"features": [ ... ]}`, into one `FeatureCatalog` whose
   `.features` is sorted by `id` (deterministic, K10). A DUPLICATE id across files
   → `CatalogError`; a missing dir → `CatalogError`; malformed yaml → `CatalogError`.
3. **Module integrity.** When `known_modules` is supplied (the real
   `custodex` module set, from `inventory`-style discovery), any feature
   naming a non-existent module → `CatalogError` (catches typos at load).
4. **Accessors.** `by_id` (loud on miss, K8) and `by_subsystem` (deterministic
   grouping). `render_features_md(catalog)` returns the human golden-ref markdown
   (grouped by subsystem, sorted, with demo/test columns) — pure (K10).
5. **Real seed loads.** A real `feature-doc/catalog/` seed (≥1 subsystem file)
   loads under the real module set with zero errors (integration).

## Design
- **`custodex/featurecatalog.py`** (new): see ARCHITECTURE.md pin.
  - `FEATURE_ID_RE = re.compile(r"^FEAT-[A-Z][A-Z0-9]*-\d{3}$")`.
  - `Feature(BaseModel)` — `model_config = ConfigDict(extra="forbid", frozen=True)`;
    field validators: id pattern, non-empty `modules`. Tuple fields for hashable
    frozenness (mirror `schema.py`/`extract.Symbol`).
  - `FeatureCatalog(BaseModel)` — `features: tuple[Feature, ...]`; `by_id`,
    `by_subsystem`.
  - `load_catalog(catalog_dir, *, known_modules=None)` — sorted `glob("*.yaml")`,
    `yaml.safe_load` each, validate, collect; raise on duplicate id / unknown
    module / empty dir-with-no-yaml / unreadable. Pydantic `ValidationError`
    wrapped into `CatalogError` with the offending file in the message.
  - `render_features_md(catalog)` — pure string build.
- **`errors.py`**: add `CatalogError(CodeDocMonitorError)` + to `__all__`.
- **`feature-doc/catalog/`** (new dir): a seed yaml (e.g. `extract.yaml`) with a
  couple of real, correct features so the loader has a real fixture to load.

## Schema / migration (K6)
None — `featurecatalog.py` is a NEW, self-contained module; no change to any
public wire schema (`ReviewRecord` etc.) or stored fingerprint. `CatalogError` is
an additive error type.

## Test plan (`tests/test_featurecatalog.py`, new — TDD red first)
- **unit:** Feature accepts a valid dict; bad id pattern → CatalogError/Validation;
  empty modules → error; extra key → error (extra=forbid); status enum enforced.
  `FeatureCatalog.by_id` returns / raises CatalogError on miss; `by_subsystem`
  groups deterministically. `render_features_md` is pure & stable (same input →
  byte-identical), contains each id + subsystem heading.
- **integration:** `load_catalog(tmp_dir)` over two yaml files aggregates + sorts;
  duplicate id across files → CatalogError naming the id; unknown module ref (with
  `known_modules`) → CatalogError; missing dir → CatalogError; a dir with no yaml →
  CatalogError (loud, not silent-empty, K8). Loads the REAL `feature-doc/catalog/`
  seed against the real `custodex` module set with no error.
- **coverage:** exercise every validator + every raise branch for ≥90% branch.

## Dogfood
`featurecatalog.py` is a new public module → it will show up as undocumented in
`cdx coverage` and (once added to the dogfood config in a later slice) drift the
api docs. For R-01 it is NOT yet added to the dogfood doc config, so the existing
`cdx check`/`lint` stay green; the self-coverage `--fail-under 95` gate may need
the new module's public symbols waived OR a stub api doc — handle minimally:
prefer adding it to the dogfood coverage waiver with reason "EPIC R, documented in
feature-doc (R-07 wiki)" so we don't prematurely hand-author an api doc the R-07
wiki will own. Confirm `cdx check` + `cdx lint` + `cdx coverage --fail-under 95`
exit 0.

## Out of scope
Traceability matrix (R-03); demo tags (R-04); test reorg/annotation (R-05/06);
source index/wiki (R-06/07); `cdx wiki` CLI (R-08); populating the full catalog
(R-02). R-01 is the loader + schema + a seed fixture only.

## Constraints
K0 (no new dep — pydantic+pyyaml already core), K1 (pure load, no side effects, no
target import), K6 (no wire/schema change), K8 (CatalogError loud on every malformed
input — never a silent empty catalog), K10 (deterministic: sorted files, sorted
features, clock-free render).
