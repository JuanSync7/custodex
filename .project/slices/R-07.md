# Slice R-07 ‚Äî Source index + source wiki (`srcindex.py`)

EPIC R, phase R6. Indexes every public symbol of `code_doc_monitor` and ties each
module to the catalog features it implements, producing the source wiki and the
SOURCE view of traceability ‚Äî and finally realizing the deferred R-02 "no orphan
public capability" check.

## Goal (validable)
1. **Index.** `build_source_index(pkg_root, catalog)` returns a `SourceIndex` of
   `ModuleIndex(module, path, public_symbols, features)` ‚Äî public symbols from
   `inventory.discover_symbols` (no AST re-impl), features joined from the catalog
   by module name. Sorted, deterministic (K10).
2. **Completeness accessors.** `features_without_module_match()` (a feature naming a
   module not in the package ‚Äî should be EMPTY for the real tree) and
   `modules_without_feature()` (a public module with no catalog feature ‚Äî the orphan
   check). Both deterministic.
3. **Real tree.** Over `code_doc_monitor`, `features_without_module_match()` is
   empty; `modules_without_feature()` lists ONLY genuinely-not-a-feature modules
   (e.g. `errors`/`__init__`-style or the EPIC-R machinery `featurecatalog`/
   `traceability`/`testwiki`/`srcindex` themselves, which are catalogued under the
   `reference` subsystem ‚Äî verify and reconcile so the orphan list is empty OR
   every entry is justified).
4. **Render.** `render_source_wiki_md(index)` is a pure, byte-stable per-module wiki
   (path, public symbols, implementing-feature links) + a coverage summary.

## Design
See ARCHITECTURE.md `srcindex.py` pin. Reuse `inventory.discover_files`/
`discover_symbols` + `featurecatalog`. Frozen pydantic, sorted, no new dep (K0),
pure (K1 ‚Äî the only target touch is the AST inventory that already exists).

## Test plan (`tests/unit/test_srcindex.py` + a `tests/system/` completeness test)
- **unit:** `build_source_index` over a tiny fixture pkg + a fixture catalog ‚Äî
  modules carry their public symbols + joined features; `features_without_module_match`
  catches a catalog feature naming a missing module; `modules_without_feature` catches
  an un-catalogued module; deterministic. `render_source_wiki_md` pure/stable.
- **integration/system:** over the REAL `code_doc_monitor` + real catalog ‚Äî
  `features_without_module_match() == ()`; assert `modules_without_feature()` is empty
  (after reconciling: every public module has ‚â•1 catalog feature, including the
  reference-subsystem EPIC-R modules). Tag this test file `Features: FEAT-REFERENCE-‚Ä¶`.
- Full gate green; add `srcindex.py` to the dogfood coverage waiver; `cdmon check`/
  `lint`/`coverage --fail-under 95` exit 0.

## Reconciliation note
If `modules_without_feature()` over the real tree is non-empty, ADD honest catalog
features for those modules (most likely: extend the `reference` subsystem to cover
`traceability`, `testwiki`, `srcindex`; decide whether `errors`/`blocks`/`generate`
etc. already have features or need one). The orphan list MUST end empty so the
golden reference provably covers the whole public surface ‚Äî that is this slice's
real value.

## Out of scope
The `cdmon wiki` CLI + `--check` + the traceability CI gate (R-08); rendering to
disk (R-08 owns regeneration). This slice ships the engine + the completeness
guarantee.

## Constraints
K0 (no new dep; reuse inventory/coverage), K1 (pure; AST inventory only), K8 (loud
on a bad pkg root via inventory's InventoryError; orphan modules surfaced, not
hidden), K10 (deterministic sorted index + render).
