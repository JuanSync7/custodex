# Slice M-02 — wire demo into central + standalone, subdir-aware git sync, e2e

Part of EPIC M. Makes the M-01 demo fully functional in BOTH the central dashboard
and the standalone view, fixes git-sync for a repo-subdir, fixes the per-unit
`.rpt` waiver math, and adds the system/e2e tests. See `.project/spec/CONFIGV2.md`
§4, §6.

## Goal (validable)

The demo is live and functional end to end: `scripts/seed_demo.py` registers the
demo (`demo-taskflow`, `local_path = <repo>/demo`) and pre-syncs it so the central
dashboard on :33333 shows its Documents relationship view + a working Sync button;
`cdx serve` from `demo/` gives the standalone view of the same. Git-mode sync
works for a config that lives in a SUBDIR of a git repo. The per-unit `.rpt`
percent is consistent with the overall percent (waivers removed from both). Proven
by a system/e2e test that drives the demo through the central app AND the
standalone app, plus a git-mode sync test against a committed dir-layout repo.

## In scope

### 1. configsync subdir-awareness (`custodex/configsync.py`)
- git mode currently assumes `config/cdmon` is at the worktree root. Make it
  subdir-aware: compute the git toplevel (`git -C local_path rev-parse
  --show-toplevel`) and `rel = local_path.relative_to(toplevel)`; create the
  detached worktree of the toplevel at `<branch>`, then read config from
  `<worktree>/<rel>/config/cdmon` with repo_root `<worktree>/<rel>`. local mode
  unchanged (reads `local_path` directly). When `local_path` IS the toplevel,
  `rel == "."` and behavior is identical to today (assert via existing tests).
- If `<branch>` does not contain `<rel>/config/cdmon` (e.g. the demo isn't
  committed to main yet), raise a clear, loud `SyncError` (K8) — the caller
  (standalone/seed) already treats git as best-effort and skips.
- Add a test: a committed git repo whose `config/cdmon` is under a subdir →
  git-mode sync reads it correctly; toplevel-root repo still works.

### 2. per-unit `.rpt` waiver fix (`custodex/report.py`)
- Per-unit `scanned`/`percent` must remove waived files from the denominator the
  SAME way the overall summary does (a unit of 3 files with 1 waived + 2
  documented reports `percent: 100.0`, not 66.67). Keep overall behavior. Update
  the N-04 tests + regenerate `demo/config/cdmon/coverage.rpt` (must stay
  idempotent). Add/adjust a test asserting per-unit waiver removal.

### 3. Central wiring (`scripts/seed_demo.py`)
- Register the demo repo: `repo_id = "demo-taskflow"`, RepoIdentity with
  `local_path = <abs path to repo>/demo`, `default_branch = "main"`, registered
  OPEN (no token) so the Sync button works token-less. Run a LOCAL
  `configsync.run_sync` (working tree — works now; git would require the demo
  committed to main) + `replace_config` + `add_sync_run` so Documents +
  sync-state are populated on first load. Keep it import-safe
  (`build_seeded_store` stays callable from tests, deterministic `_NOW`). Don't
  break the existing 4 demo repos.

### 4. e2e / system tests (`tests/test_demo_e2e.py`, new)
- **central**: via TestClient over the seeded app — assert `demo-taskflow` is
  listed; `GET /repos/demo-taskflow/documents?sync_kind=local` returns the
  doc→code_refs tree matching `demo/config/cdmon`; `POST /sync {mode:"local"}`
  token-less succeeds and reports `fully_synced` + counts; `GET /sync-state`
  returns the run.
- **standalone**: `build_standalone_app(<repo>/demo, ...)` via TestClient — one
  repo, documents tree, token-less sync, `GET /` 200.
- **git-mode (committed fixture)**: build a SEPARATE temp git repo (copy a
  minimal dir-layout, git init + commit on main) whose `config/cdmon` is in a
  SUBDIR; `run_sync(mode="git")` reads main correctly; no leftover worktrees.
- **demo CLI smoke**: from `demo/`, `cdx check` exit 0 + `cdx rpt` matches the
  committed `coverage.rpt` (or regenerate-and-compare).
- ≥90% branch coverage of changed engine code; full suite green.

### 5. Demo README
- Update `demo/README.md` with the central + standalone instructions and note
  that git-mode sync requires the demo committed to the repo's default branch
  (local-mode works against the working tree today).
