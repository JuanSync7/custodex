# Slice L-01 — `cdmon serve`: standalone per-repo view

Part of EPIC L. Builds on Y-02 (sync engine/endpoints) + W (dashboard). The
no-central-access app: a user runs it in their own repo and gets the SAME
Documents/relationship/Sync dashboard scoped to ONLY that repo. See
`.project/spec/CONFIGV2.md` §5.

## Goal (validable)

`cdmon serve [--host 127.0.0.1] [--port 0] [--repo-id ID] [--no-open]` launches
the FastAPI app + dashboard SPA backed by an InMemoryStore in which ONLY the
current repo is auto-registered (`local_path = cwd`, `default_branch = main`,
registered OPEN — no token) and immediately git+local synced so the Documents
relationship view + sync-state are populated on first load. The fleet view shows
exactly one repo; its Documents page + Sync buttons (main/local) work with no
token. Proven by an import-safe builder + e2e test (TestClient): the app lists
exactly the one repo, `GET /documents?sync_kind=git` returns its doc→code_refs
tree, `POST /sync` works WITHOUT a bearer token, and `GET /` serves the SPA (or
the landing JSON when `dist/` is absent).

## In scope

### Backend — `code_doc_monitor/server/standalone.py` (new) or extend seed_demo
- `build_standalone_store(repo_root: Path, *, repo_id: str | None, now: str) ->
  InMemoryStore`: register the repo OPEN (RegistrationPayload with the
  RepoIdentity carrying `local_path=str(repo_root)`, `default_branch="main"`, NO
  auth_token), then run `configsync.run_sync` for BOTH modes (git best-effort —
  if the repo has no `main`/commits, skip git gracefully; local always) and
  `replace_config` + `add_sync_run` so the views are pre-populated. `repo_id`
  defaults to the bundle index `repo` (else dir name). Import-safe (no server
  launch), mirroring `seed_demo.build_seeded_store`.
- `build_standalone_app(repo_root, *, repo_id, now) -> FastAPI`: `create_app`
  over that store + serve the dashboard SPA exactly like `seed_demo`
  (`create_seeded_app`) — reuse that SPA-mounting helper, don't duplicate it.
- `cdmon serve` in cli.py: resolve repo_root = cwd; build the app; `uvicorn.run`
  on host/port (port 0 = auto). Print the URL. Loud K8 if cwd has no
  `config/cdmon/index.yaml` (tell the user to `cdmon init --v2`). Keep launch
  thin; all logic in the import-safe builders (so tests never bind a socket).

### Frontend — small SyncControls relaxation (`dashboard/src/components/SyncControls.tsx`)
- Allow a sync attempt with NO token (standalone/open repos): the button is NOT
  hard-guarded on an empty token. `syncRepo` omits the `Authorization` header
  when the token is empty (adjust `api/client.ts` syncRepo accordingly). On a
  401/403 response, THEN reveal the token input + auth message (central repos).
  Keep the existing behavior for repos that do require a token (the input still
  appears after the auth error). Update SyncControls tests: empty-token click
  against an open repo SUCCEEDS (no input shown); 403 reveals the input.

## Test plan

- `tests/test_standalone.py` (new): build a real temp git repo + config/cdmon +
  in-sync docs; `build_standalone_store` → `list_repos()` has exactly 1 repo with
  `local_path` set + a populated `config_documents_for`/`latest_sync_run`;
  `build_standalone_app` via TestClient → `GET /repos` one repo; `GET
  /repos/{id}/documents?sync_kind=git` returns the tree; `POST /repos/{id}/sync
  {mode:"local"}` succeeds with NO Authorization header (open repo); `GET /`
  returns 200 (SPA or landing JSON). Cleanup: no leftover git worktrees.
- Frontend: updated `SyncControls.test.tsx` (empty-token success path; 403 still
  reveals input). `npm run test:run` + `npm run build` pass.
- ≥90% branch coverage of new backend code; full python suite green.

## Out of scope
Demo content (M-01) — L-01 is the mechanism; the demo USES it. Actual socket
binding in tests (use TestClient/import-safe builders only).

## Constraints
K1 (serve reads the repo; sync uses the read-only git-worktree path), K8 (loud on
missing config), K10 (injected clock; deterministic). Reuse seed_demo's SPA mount
+ create_app — do NOT fork them. Offline (no network; git is local).
