# Slice P-01 — Extractor protocol + flag-gated body-AST dimension

EPIC P. Adds (a) a pluggable **Extractor protocol + registry** with today's Python
AST extractor as the default behind the seam (pure refactor, K0), and (b) an
opt-in **body-AST fingerprint tier** so an eng-guide can detect an implementation
change a signature/docstring hash misses — while holding the **user-guide
signature-hash bytes constant** (K3) and keeping every stored `cdm.fingerprint`
valid when the flag is OFF (the default).

## Goal (validable)
1. **Pure-refactor seam.** `Extractor` Protocol + language-keyed registry exist;
   `extract_file` is a thin wrapper over `get_extractor("python")`. Proven by a
   regression test asserting frozen golden `surface_hash()` literals
   (`user-guide 9e9171f520832497`, `eng-guide f688a396c2e8be27`) over the test
   `SAMPLE`, AND `get_extractor("rust")` raising `ExtractionError` (K8).
2. **Flag-gated body tier.** Flag OFF (default) → `surface_hash()` bytes identical
   to golden for both audiences. Flag ON → an eng-guide surface over a function
   whose body changed (signature/docstring unchanged) yields a DIFFERENT hash; the
   same change for a user-guide yields the SAME hash. Body hash insensitive to
   comments/formatting/docstring and deterministic (K10). Proven by a three-case
   test (OFF-invariance / ON-eng-moves / ON-user-stable).

## Design
- **extract.py — Extractor seam.**
  - `@runtime_checkable class Extractor(Protocol)` with `language: str` and
    `extract(self, path: Path) -> list[Symbol]` (mirrors `registry.py` /
    `heal.ProposedFixLike`; stdlib `typing` only, K0).
  - `class PythonAstExtractor` (`language="python"`) wrapping the existing
    `extract_file` body, moved verbatim into private `_extract_python_symbols`.
  - `_EXTRACTORS: dict[str, Extractor] = {"python": PythonAstExtractor()}`;
    `register_extractor(extractor)`; `get_extractor(language)` loud on unknown (K8).
  - `extract_file(path)` → `get_extractor("python").extract(path)` (callers
    untouched). Add the four names to `__all__`.
- **extract.py — body tier.**
  - `Symbol.body_hash: str | None = None` (functions/methods only; None for
    class/variable). Frozen-safe (new field, default).
  - `_body_ast_hash(node)` — strip a leading docstring Expr stmt, then
    `sha256("\x00".join(ast.dump(s, include_attributes=False) for s in body))[:16]`.
    Populated in `_func_symbol`.
  - `surface_hash(self, *, include_body: bool = False)` — additive key, same
    discipline as the existing `records` key:
    `include_body_tier = include_body and self.audience is not Audience.USER_GUIDE`;
    emit `entry["body_hash"]` only when `include_body_tier and sym.body_hash`. OFF →
    byte-identical to today; ON → user-guide invariant, class/var-only eng-guide
    invariant.
- **config.py — opt-in flag.** `MonitorConfig.fingerprint_body_tier: bool = False`;
  mirror onto CONFIG-V2 `IndexFile`; lift in `load_bundle`'s `base_config` build.
  Additive `False` default → existing configs + stored fingerprints stay valid.
- **One-shared-truth threading.** Pass `config.fingerprint_body_tier` as
  `include_body=` at EVERY `surface_hash()` call that decides identity:
  `drift.py` (the HASH comparison), `heal.py` + `layout.py` (the `set_fingerprint`
  stamps), `monitor.py` (record `surface_hash`/`new_record_id`), `cli.py` (display
  parity). Heal-stamp and detect MUST agree or a permanent self-inflicted HASH
  drift results (the B-03 lock-predicate lesson, generalized).

## Schema / migration (K6)
P1 does NOT bump `ReviewRecord.schema_version`: the public wire shape is unchanged
(`surface_hash` is still a 16-hex string; only its derivation gains an opt-in
tier). The surface-contract change (`Symbol.body_hash`, gated `surface_hash`) is
pinned in ARCHITECTURE.md (done). Add `fingerprint_body_tier` to the
`config_snapshot` dict so a record self-describes its derivation (free-form dict
key). The genuine structured-fingerprint shape change + `schema_version` bump is
**P2**. A migration note is appended to LESSON_LEARNT.md.

## Test plan
- **unit (test_extract.py):** registry resolve/`isinstance(Extractor)`/loud-unknown;
  golden byte-invariance (both audiences); body hash insensitive to
  comments/formatting; changes on logic edit; ignores docstring; None for
  class/var; deterministic; recurses nested defs; decorator change does NOT move
  body_hash (boundary); async body change moves it; flag gating
  (OFF==golden / user-guide ON==OFF / eng-guide ON moves on body-only /
  eng-guide class-var-only ON==OFF).
- **integration (test_config.py, test_drift.py):** flag defaults False + config
  omitting it loads; eng-guide body-only edit → no drift OFF, HASH drift ON;
  user-guide body-only edit ON → never drift; heal-stamp ON + detect ON → clean.
- **system/e2e (test_system.py):** flag absent → `cdx check` clean across a
  comment-only edit; `fingerprint_body_tier: true` → eng-guide body edit drifts,
  `monitor --apply` re-heals clean + idempotent (K7), sibling user-guide stays
  clean.
- Additive: all prior tests green; ≥90% on changed code; gate ≥90% (`--cov-branch`).

## Dogfood
`extract.py`/`config.py`/`drift.py`/`heal.py`/`layout.py`/`monitor.py`/`cli.py`
are tracked (eng-guide). Flag defaults OFF → the fingerprint reheal is a no-op for
the body tier; the reheal that fires is ordinary signature drift on
`docs/api/pipeline.md` from the new public symbols (`Extractor`,
`PythonAstExtractor`, `register_extractor`, `get_extractor`). Reheal with
`cdx monitor --apply --config config/cdmon`; confirm `cdx check` + `cdx lint`
exit 0. ARCHITECTURE signatures pinned first (done).

## Out of scope
The structured `SurfaceFingerprint` + which-tier-moved drift reporting + the
`schema_version` bump (P2); non-Python extractors (P3); region anchors (P4).
P1 only introduces the seam (sole Python path) and the opt-in body tier.
