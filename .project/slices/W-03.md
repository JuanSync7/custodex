# Slice W-03 — dashboard Sync button (git / local) + sync-state badge

Part of EPIC W. Builds on Y-02 endpoints + W-01. See `.project/spec/CONFIGV2.md`
§5. Also fixes the W-01 repo-id encoding inconsistency.

## Goal (validable)

A `<SyncControls>` component on the repo pages shows the current sync state
(fully_synced badge, commits_ahead, drift count, last-synced time) and offers two
actions — **Sync (main)** → `POST /sync {mode:"git"}` and **Sync (local)** →
`POST /sync {mode:"local"}` — using the per-repo bearer token. After a sync it
refreshes the state from the returned run and re-loads the Documents view.
Proven by Vitest: initial state (incl. the never-synced `null` case), clicking
each button posts the right mode + token and renders the returned summary, and
the auth-error path surfaces a message.

## API contract (captured)

- `POST /api/repos/{id}/sync` body `{"mode":"git"|"local"}`, header
  `Authorization: Bearer <token>` → 201 with a SyncRun:
  `{sync_kind, fully_synced, commits_ahead, document_count, code_ref_count,
    drift:{ok,drift_count,by_kind:{KIND:count},coverage_percent}, branch,
    head_commit, main_commit, ref, started_at, finished_at, repo_id}`.
  401 no token / 403 wrong token / 404 unknown / 400 bad mode or no local_path.
- `GET /api/repos/{id}/sync-state?sync_kind=` → a SyncRun or `null` (never synced).

## In scope — `dashboard/src/`

- **Fix encoding (from W-01):** make `documentsFor` use the SAME slash-preserving
  `encodeRepoId` helper as the other client methods (NOT `encodeURIComponent`).
  Update its client test to the slash-preserving expectation. (Repo ids can be
  `org/name`; the server route is `{repo_id:path}`.)
- `types.ts`: `SyncRun` interface matching the contract; `SyncMode = "git"|"local"`.
- `api/client.ts`:
  - `syncRepo(repoId, mode: SyncMode, token: string): Promise<SyncRun>` →
    POST `.../repos/{enc}/sync` with `{mode}` + Bearer header (match the existing
    `resolve()` write-path auth pattern).
  - `syncState(repoId, syncKind?): Promise<SyncRun | null>` →
    GET `.../repos/{enc}/sync-state` (+ `?sync_kind=` when given).
  - Client tests for both URLs/headers.
- `components/SyncControls.tsx`: props `{repoId, api?, token?, onSynced?}`.
  Loads `syncState` on mount (handle `null`). Renders a status row: a
  sync/drift/review dot (reuse `.dot--*`), "In sync" vs "N commits ahead" vs
  "M drift", coverage %, last-synced (finished_at). Two buttons (git/local) with
  a busy state; a token input if no token prop (reuse RepoDetail's token UX). On
  click → `syncRepo`, update state from the result, call `onSynced`. Error
  (ApiError) → inline message (401/403 → "auth required/invalid").
- Mount `<SyncControls>` in the repo header area — on `RepoDetail.tsx` and on
  `Documents.tsx` (pass `onSynced` to re-fetch the documents list). Keep it a
  single reusable component.
- `index.css`: status-row + button styling reusing existing tokens/dots.
