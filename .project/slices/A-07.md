# Slice A-07/A-08 — gap→owner suggestions + coverage manifest writer

Part of EPIC A (final slices). Builds on A-01→A-06. Two deliberate engineering
calls, flagged for the user's review:

> **Decision 1 (A-07):** the gap→owner suggester is a **deterministic heuristic**
> in `coverage.py`, NOT routed through the `Backend` Protocol. Rationale: the
> `Backend.propose(FixRequest) -> BackendResult` interface is shaped for *drift
> remediation*; widening it (a new method on all four backends) for a coverage
> concern violates the "conform to the narrowest interface" discipline
> (LESSON CDM-11). A heuristic is offline + deterministic by construction (honors
> the roadmap's "mock proposes a deterministic mapping" intent). An LLM-enhanced
> suggester can be added later as an optional, separate capability.

> **Decision 2 (A-08):** the persistent index is written to a **dedicated,
> regenerable manifest** (default `.cdmon/coverage.json`), NOT injected as a
> subsection into the hand-authored `cdmon.yaml`. Rationale: rewriting
> `cdmon.yaml` would destroy its comments (comment-preserving YAML needs
> `ruamel.yaml` = a new dependency, violating K0), and `MonitorConfig` is
> `extra="forbid"` so an auto-generated key wouldn't round-trip. A separate
> generated artifact is lossless, idempotent (K7), and keeps the config
> hand-owned. (If the user still wants it *inside* the yaml later, that's a
> ruamel-based follow-up slice with K0 amended.)

## Goal (validable)

(A-07) `suggest_owners(report, config)` returns, for each undocumented public
symbol/file, a deterministic suggested owner: an **existing** doc id when that
doc already owns a sibling in the same file, else a **proposed new doc id**
derived from the module path — proven by a fixture test asserting exact
suggestions. (A-08) `cdx coverage --write [PATH]` writes a deterministic
coverage manifest; re-running with no change rewrites nothing and reports "no
change" (idempotent, K7) — proven by a write-then-write-again test.

## In scope

### A-07 — `coverage.py`: `suggest_owners` (pure, deterministic)
```python
class OwnerSuggestion(BaseModel):     # frozen, extra="forbid"
    path: str
    name: str | None                  # None for a whole-file suggestion
    suggested_doc_id: str             # existing or proposed
    is_new_doc: bool                  # True => proposed new doc
    reason: str                       # why (sibling-owned / new-module)

def suggest_owners(report: CoverageReport, config: MonitorConfig) -> tuple[OwnerSuggestion, ...]: ...
```
Heuristic (deterministic, K10):
- For an undocumented public symbol in file F: if any document already owns
  another symbol in F, suggest **that** doc id (lowest doc id if several),
  `is_new_doc=False`, reason "sibling symbols already in <doc>".
- Else (F entirely unowned): suggest a new doc id derived from F's module path
  (e.g. `pkg/sub/mod.py` → `mod` or `pkg-sub-mod` — pick one, document it),
  `is_new_doc=True`, reason "no doc references <F>". Group all of F's gaps under
  the same proposed id.
- Sorted output (path, then name). Pure, no I/O, no LLM, no new dep.

### A-08 — `cdx coverage --write [PATH]` (opt-in mutation)
- New flag `--write [PATH]` (default `.cdmon/coverage.json`) on the `coverage`
  command. It writes a deterministic manifest = the full report payload (reuse
  A-05's `_coverage_payload` so baskets/percentages are included) PLUS the A-07
  `suggestions`. Sorted keys, stable formatting (K10).
- Idempotent (K7): read the existing manifest first; if the new content equals
  it, do NOT rewrite and print "coverage manifest unchanged"; else write and
  print the path. (Mirror `heal.regenerate_regions`' changed/unchanged return.)
- `--write` is the only mutating mode; the default `coverage` invocation stays
  read-only (K1). `--write` + `--json` together: write the file (the `--json`
  human-vs-json toggle governs stdout only).

## Test plan
- **A-07** (`tests/test_coverage.py`): fixture with (a) a file partly owned →
  sibling-suggestion to the existing doc; (b) a fully-unowned file → new-doc
  suggestion grouping its gaps; deterministic order; private gaps not suggested.
- **A-08** (`tests/test_cli.py`): `--write` creates the manifest (parse it back,
  assert percentages + a known gap + a suggestion present); second `--write` with
  no source change → "unchanged" + file mtime/content identical; `--write PATH`
  honors a custom path; manifest is valid JSON.
- Coverage ≥90% new code; whole-pkg gate ≥90%.
