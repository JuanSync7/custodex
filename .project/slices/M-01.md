# Slice M-01 — `demo/` adopter repo content

Part of EPIC M. Builds the self-contained demo adopter repo (a subdir of THIS
repo, using the outer git history) with a complete `config/cdmon/` dir layout.
M-02 wires it into the central + standalone + e2e. See `.project/spec/CONFIGV2.md`
§6.

## Goal (validable)

`demo/` is a small, realistic Python app fully monitored by cdx via the dir
layout. From `demo/`: `cdx check` is CLEAN (docs in sync), `cdx coverage`
reports a sensible %, and `cdx rpt --write` produces `demo/config/cdmon/
coverage.rpt`. The demo exercises the new format end to end: frontmatter,
dir-covered, source-files-format (with a non-source file present to prove it is
NOT counted), index.yaml, ignore.yaml (+ a demo `.gitignore`), doc-style.yaml +
demo writing templates. Proven by a check/coverage/rpt run from `demo/` (capture
outputs) and that `load_bundle(demo/config/cdmon)` succeeds.

## In scope (all under `demo/`)

- `demo/src/<app>/...` — a realistic small app across 2 directories, ~4–6 `.py`
  modules with genuine classes/functions + docstrings (e.g. a "taskflow" task
  engine: `core/` model+engine, `io/` storage+report). Real, coherent code (not
  stubs). Lint-clean (ruff) — if needed, add `demo/` to the ruff/mypy exclude in
  pyproject.toml WITHOUT changing existing rules.
- Include ONE non-`.py` file under a `dir-covered` dir (e.g. `core/notes.log` or
  a `.rpt`) to demonstrate `source-files-format: ['.py']` excludes it from
  coverage.
- `demo/docs/api/*.md` — eng-guide docs with managed CDM regions referencing the
  modules (so `cdx monitor --apply` can heal them in-sync). An index/landing
  doc is optional.
- `demo/config/cdmon/`:
  - `index.yaml` — frontmatter (repo e.g. `demo-taskflow`), `root: "../.."`,
    backend mock, global region_templates if needed, coverage.waive for
    `__init__.py`, `units:` listing the unit files, ignore + doc-style pointers.
  - 2 unit `*.yaml` — frontmatter + dir-covered + source-files-format `[".py"]` +
    documents (DocumentSpec with code_refs). Disjoint dir-covered.
  - `ignore.yaml` — `gitignore: true` + patterns (incl. `*.rpt`, `*.log`,
    `__pycache__`).
  - `doc-style.yaml` — defaults + per-doc mappings.
  - `coverage.rpt` — GENERATED via `cdx rpt --write` (commit the result).
- `demo/templates/writing/{document-type,tone,writing-style,vocabulary}/*.md` —
  the writing templates the demo's doc-style.yaml references (reuse the canonical
  content from the repo `templates/writing/`, copied so the demo is
  self-contained — its repo_root is `demo/`).
- `demo/.gitignore` — a couple of patterns so `ignore.yaml gitignore:true` has a
  real source.
- `demo/README.md` — how to drive the demo: `cd demo && cdx check`,
  `cdx coverage`, `cdx rpt --write`, `cdx serve` (standalone), and a note
  that M-02 registers it in the central demo on :33333.
