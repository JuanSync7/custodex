# Slice Y-03 — `cdx sync` CLI (local + remote)

Part of EPIC Y. Builds on Y-02 (engine + endpoints already added local_path).
The client-facing trigger for a sync. See `.project/spec/CONFIGV2.md` §4.

## Goal (validable)

`cdx sync [--mode git|local] [--remote URL --repo-id ID] [--json]` runs a
config sync. With NO `--remote` it runs `configsync.run_sync` against the current
repo (cwd as local_path) and prints the summary (fully_synced, document/code_ref
counts, commits_ahead, drift summary) — the offline path a per-repo user without
central access uses. With `--remote URL --repo-id ID` it POSTs to
`{URL}/repos/{ID}/sync` (bearer token from the central token env, like the
existing central client) and prints the server's run summary. Proven by: a CLI
test over a real temp git repo asserting the local printout (and `--json` shape)
matches `run_sync`; and a remote test against a FastAPI TestClient asserting the
POST path returns + prints the server summary.

## In scope — `custodex/cli.py`

- `cdx sync`:
  - `--mode {git,local}` (default `local`).
  - `--remote URL` (optional), `--repo-id ID` (required iff `--remote`),
    `--token-env` (default the existing `DEFAULT_CENTRAL_TOKEN_ENV`).
  - `--default-branch` (default `main`), `--json`.
  - Local path: `run_sync(Path.cwd(), repo_id or <derived>, mode=...,
    default_branch=..., now=<injected clock>)`; print human summary or `--json`
    (the SyncRun as JSON via the public schema). Exit 0; non-zero only on a loud
    error (K8). `repo_id` when local-only defaults to the index `repo` field (or
    the dir name) — read it from the bundle, don't invent.
  - Remote path: reuse the repo's existing HTTP client helper (find how
    `cdx register` POSTs to the central server — reuse that requests/urllib
    seam + auth) to `POST {URL}/repos/{repo_id}/sync` with `{mode}`; print the
    returned summary. Loud K8 on HTTP error / missing token.
- Reuse the injected clock seam (`cli._now`) for determinism.

## Test plan (`tests/test_cli_sync.py`, new)

- **local**: temp git repo + config/cdmon + in-sync docs; `cdx sync` prints
  fully_synced + counts; `--json` parses to a SyncRun-shaped dict equal to the
  engine's; `--mode git` works; an edit → drift shown; exit codes correct.
- **remote**: spin a FastAPI TestClient (or monkeypatch the HTTP seam) with a
  registered repo (local_path set); `cdx sync --remote ... --repo-id ...`
  triggers the POST and prints the server summary; missing token / HTTP 4xx →
  loud non-zero exit.
- **loud**: unknown mode; not a git repo (when git info required) handled with a
  clean message (no traceback, K8).
- ≥90% branch coverage of new code; full suite green.

## Out of scope
Frontend (W), standalone serve (L-01). The DB persistence happens server-side on
the remote path (Y-02); the local path just computes + prints (no local DB).

## Constraints
K8 (loud, no traceback), K10 (injected clock), K1 (sync reads; never mutates the
working tree). Offline tests (TestClient / monkeypatched HTTP — no real network).
