# Slice A-02 — symbol-level inventory

Part of EPIC A. Builds on A-01 (`inventory.discover_files`). Attaches the public
(and private, for losslessness) symbol surface to each inventoried code file, so
A-03 can compute per-symbol ownership.

## Goal (validable)

Given an `Inventory` (from A-01) over a real fixture mini-repo, `discover_symbols`
returns, for every Python file, the exact set of its symbols (name + kind +
is_public) as produced by `extract.extract_file`, in deterministic order; a
non-Python file (e.g. `language != "python"`) appears with an empty symbol tuple
(tracked, not dropped — losslessness). Proven by a test asserting the exact
per-file symbol names on a fixture.

## In scope — extend `code_doc_monitor/inventory.py` (pure; reuse `extract`)

Proposed contract (finalize + add to ARCHITECTURE.md `## inventory.py`):

```python
class FileSymbols(BaseModel):       # frozen, extra="forbid"
    path: str                       # repo-relative POSIX (matches CodeFile.path)
    language: str
    symbols: tuple[Symbol, ...]     # extract.Symbol; () for non-python / no symbols

class SymbolInventory(BaseModel):   # frozen, extra="forbid"
    root: str
    files: tuple[FileSymbols, ...]  # same order as Inventory.files

def discover_symbols(inventory: Inventory, root: Path) -> SymbolInventory: ...
```

- Reuse `extract.extract_file(root / path)` per **python** file; non-python files
  get `symbols=()`. Do NOT re-implement AST parsing.
- Deterministic order (K10): files in `Inventory.files` order; symbols in
  `extract_file`'s existing order.
- **Unparseable-file policy (decide + document):** for this slice, let
  `extract.ExtractionError` propagate (loud, K8) — a syntax-error file aborts the
  scan. Note in LESSON_LEARNT that a future `--skip-unparseable` resilient mode
  may be wanted for scanning arbitrary external repos (EPIC G), and capture it in
  `.project/problems/A-02.md` if you think it's a real adoption blocker.
- Keep `Symbol` import light; no new dependency (K0). No FS mutation (K1).

## Test plan (`tests/test_inventory.py`, extend)

- **unit/integration on a real fixture**: mini-repo with `pkg/a.py` (a public
  function + a private `_helper` + a class with a method), `pkg/sub/b.py`, and a
  non-python file that still matched an include (give it a matched extension via
  custom include, e.g. include `("**/*.py","**/*.txt")` and assert the `.txt`
  file has `symbols=()` and `language="unknown"`). Assert exact symbol-name sets
  and `is_public` flags per file, and SymbolInventory order == Inventory order.
- **determinism**: same tree → identical `SymbolInventory`.
- **loud**: a fixture python file with a syntax error → `ExtractionError`.
- Coverage of the new code ≥ 90% (aim 100%); whole-pkg gate ≥ 90%.

## Out of scope

Ownership/coverage math (A-03), config `coverage:` (A-04), CLI (A-05). Do not
wire into cli/config/monitor.
