# Slice A-04 ‚Äî config `coverage:` section + waivers

Part of EPIC A. Adds the **index between code and docs**: a `coverage:` block in
the config carrying the scan scope (include/exclude globs) and the explicit
**waive** list (code that intentionally needs no doc, with a reason). Folds
waivers into `CoverageReport`. Builds on A-03.

## Goal (validable)

A `cdmon.yaml` with a `coverage:` section round-trips through `load_config`; a
waiver entry (`{path: glob, symbol: name?, reason: str}`) moves matching items
out of the gap basket into a new **waived** basket and removes them from the
percentage *universe* (so a fully-waived-or-documented repo reports 100%); a
waiver missing its `reason` raises `ConfigError` (K8). Proven by: a round-trip
test, and a resolver test asserting a waived gap leaves `undocumented_*`, enters
`waived_*`, and lifts the percentage.

## In scope

### `config.py` ‚Äî new models (additive; defaults keep old configs valid)
```python
class WaiverEntry(BaseModel):       # frozen, extra="forbid"
    path: str                       # glob over repo-relative POSIX paths
    symbol: str | None = None       # exact symbol name; None => whole-file waiver
    reason: str                     # REQUIRED (K8: a waiver must justify itself)

class CoverageConfig(BaseModel):    # frozen, extra="forbid"
    include: tuple[str, ...] = inventory.DEFAULT_INCLUDE
    exclude: tuple[str, ...] = inventory.DEFAULT_EXCLUDE
    waive: tuple[WaiverEntry, ...] = ()

class MonitorConfig(BaseModel):
    ...
    coverage: CoverageConfig = CoverageConfig()   # NEW, defaulted
```
- Importing `inventory.DEFAULT_INCLUDE/EXCLUDE` into config is fine (same
  package, no cycle ‚Äî inventory does not import config). If a cycle appears,
  redefine the literal defaults in config and assert equality in a test.
- Update `CONFIG_TEMPLATE` (the `cdx init` starter) to show a commented
  `coverage:` example. Keep the template round-tripping (existing test).

### `coverage.py` ‚Äî fold waivers into the report (additive)
Extend `resolve_coverage` (or add a thin wrapper it calls) so that, reading
`config.coverage.waive`:
- a matching `OwnedFile`/`OwnedSymbol` that is otherwise an **unowned gap** is
  reclassified **waived** (matched by glob path + optional exact symbol name;
  reuse `inventory`'s glob translation for the path);
- new `CoverageReport` members: `waived_files`, `waived_symbols` properties (or
  fields) + the waiver `reason` surfaced per waived item;
- `percent_files` / `percent_public_symbols` exclude waived items from BOTH
  numerator and denominator (universe = total ‚àí waived). A waiver that matches an
  *already-documented* item is a no-op for the basket but should still validate
  (optionally warn ‚Äî your call; document the choice).
- **Additive guarantee:** with the default empty `waive`, every A-03 test still
  passes unchanged (percentages and baskets identical). Run them to prove it.
