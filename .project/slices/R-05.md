# Slice R-05 — Test taxonomy: reorganize tests/ into clear boundaries

EPIC R, phase R4. The 62 test files live flat under `tests/`. R-05 physically
reorganizes them into a four-boundary system — `tests/unit/`,
`tests/integration/`, `tests/system/`, `tests/smoke/` — keeping `tests/regression/`,
adds per-boundary pytest markers (auto-applied by path), and a marker-lint that
fails if any test is unclassified. The full gate stays green.

## Boundary definitions (the classification rule)
- **unit** — exercises ONE module's pure logic; no cross-module orchestration, no
  CLI/server, no real-repo fixture beyond a tmp file. (e.g. schema, records,
  similar, promotion, manifest, inventory, extract-core, featurecatalog,
  traceability-core, templates, docstyle, reviewlog, sinks, ticket, issues.)
- **integration** — exercises a module-to-module SEAM or config loading across
  modules (no full CLI). (e.g. config, config_v2/index/ignore/edits, configsync,
  heal, layout, build, monitor, backends, agent, generate, pr, syncpr, report,
  doctor, apply_fix, context_refs, db, server_store_parity, server_sync,
  editable_tree, store_config.)
- **system** — drives the CLI or server end-to-end on a REAL fixture/demo/example.
  (e.g. system, cli, cli_init, cli_sync, demo_e2e, demo_walkthrough,
  demo_traceability, dirlayout_e2e, e2e_ticket_coverage, example_external,
  example_multilang, ci_templates, dogfood, server, server_launch, standalone,
  live_llm.)
- **smoke** — trivial import/availability/wiring checks. (e.g. smoke.)

Files are classified by their **dominant** boundary (the majority/primary purpose).
The PRECISE per-test boundary is annotated per function in R-06 (`Boundary:` tag),
which drives the test wiki; the directory is the coarse home. A subagent producing
the move MUST output a full per-file classification table for review.

## Goal (validable)
1. Every former `tests/test_*.py` lives under exactly one of `tests/{unit,
   integration,system,smoke}/`; `tests/regression/` unchanged.
2. **Full gate green** after the move (ruff+mypy+pytest ≥90% branch; same pass
   count as before, ± none). `cdx check`/`lint`/`coverage --fail-under 95` exit 0.
3. `pytest -m unit` / `-m integration` / `-m system` / `-m smoke` each select a
   non-empty subset; together (+ regression) they cover every collected test.
4. A **marker-lint** test asserts every collected item carries exactly one boundary
   marker (no unclassified test) — the "clear boundary" guarantee, automated.

## Safety protocol (depth-independence FIRST — order matters)
1. Add `tests/_repo.py` with `REPO_ROOT: Path` = upward search for `pyproject.toml`
   (depth-independent). Replace EVERY `Path(__file__).resolve().parents[1]` /
   `.parent.parent` that resolves to repo root with `from tests._repo import REPO_ROOT`
   (16 occurrences, ~14 files) — BEFORE moving anything, gate green.
2. Add `__init__.py` to each new boundary dir so the `tests.<boundary>.<module>`
   import path keeps working (tests is a package).
3. Fix the cross-test imports (`from tests.test_config_v2 import …` in
   test_config_index/test_docstyle/test_agent_style; `from tests.test_docstyle …`)
   to the new `tests.<boundary>.<module>` paths.
4. `git mv` each file into its boundary dir (preserve history).
5. Root `tests/conftest.py`: `pytest_collection_modifyitems` auto-adds the boundary
   marker from the item's path (`tests/unit/*` → `unit`, etc.); leave
   `tests/regression/*` with its existing `regression` marker. Preserve existing
   `pg`/`live_llm` markers on individual tests.
6. Register `unit`/`integration`/`system`/`smoke` markers in pyproject
   `[tool.pytest.ini_options].markers`. `testpaths=["tests"]` already recurses.

## Test plan
- The reorg itself is verified by the FULL suite staying green (same tests, new
  homes) + `cdx` dogfood green.
- New `tests/<unit-or-smoke>/test_boundaries.py` (marker-lint): collect-time check
  that every item has exactly one of the four boundary markers OR the regression
  marker; fail loud listing any unclassified node.
- Spot-run each `pytest -m <boundary> --collect-only -q` → non-empty.

## Dogfood
Test files are not tracked source → no api-doc drift. `tests/_repo.py` is a test
helper, not shipped. Confirm `cdx check`/`lint`/`coverage --fail-under 95` exit 0.

## Out of scope
Per-test `Boundary:`/`Feature:`/`Asserts:` annotation + the test wiki (R-06);
splitting mixed files into per-boundary files (we classify whole files by dominant
boundary — finer truth is the per-test annotation in R-06). Source index (R-07).

## Constraints
K8 (marker-lint fails loud on an unclassified test), K10 (deterministic collection),
K0 (no new dep — stdlib + pytest only). No behavior change to any test — pure move +
markers; the suite's pass set is identical before and after.
