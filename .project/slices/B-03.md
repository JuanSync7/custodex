# Slice B-03 — `llm-seeded` lock (per-region content hash)

EPIC B. Adds the mode-agnostic **lock mechanism**: a per-region content hash in
front matter that detects a human edit. `llm-seeded` = behaves like `generated`
(engine fills it) until a human edits it, then **locks** to human-owned. The same
per-region hash also fixes B-02's known limitation (the human advisory clearing
when the doc fingerprint is healed).

## Goal (validable)
A doc with `region_modes: {symbols: llm-seeded}`: (1) first heal FILLS the region
from the surface and stamps `cdm.region_hashes.symbols`; (2) after a human edits
that region body, a subsequent heal LEAVES it byte-identical (the stored
per-region hash diverges from the current body hash → locked); (3) the locked
region, when the code moves, is reported `REGION, healable=False` like a human
region. Proven by a three-phase test (fill → human-edit → re-heal-leaves-it).

## Design
- **manifest.py** — per-region hashes under `cdm.region_hashes` (a mapping id→hash):
  - `region_body_hash(body: str) -> str` — sha256[:16], normalized like the
    existing surface/`md_source_hash` (CRLF-normalized) for portability/determinism.
  - `stored_region_hash(doc, region_id) -> str | None`, `set_region_hash(meta,
    region_id, value) -> meta`. Additive to the `cdm:` mapping; `set_fingerprint`
    already copies the whole `cdm:` map, so hashes survive a heal (zero blast).
- **heal.py** — when `_corrected`/`regenerate_regions` writes a region the engine
  authored (generated OR an unlocked llm-seeded), stamp its `region_hashes[id]`
  with the hash of the written body. A region is **locked** iff it has a stored
  hash AND the current body's hash ≠ that stored hash (human edited). Locked ids
  are added to `preserve` automatically (reuse B-02's preserve machinery).
- **drift.py** — `mode_for(id)` is `LLM_SEEDED`:
  - unlocked (current body hash == stored, or no stored yet) → treat like
    `generated` (REGION healable=True, regenerate on drift).
  - locked (hash diverged) → treat like `human` (REGION healable=False when code
    moved; the "human-owned" advisory; suppress UNHEALABLE for no-renderer).
- **The lock predicate is shared** — put `is_locked(doc/meta, region_id, mode)` in
  one place (manifest or a small helper) so drift, heal, and monitor agree (the
  CDM-07 "one shared truth" rule).
- **B-02 retrofit (do it here):** a `human`-mode region can ALSO carry a
  `region_hashes` entry stamped when it was last reviewed/generated, so its
  advisory persists across a fingerprint heal until the body actually changes.
  Keep this minimal and test it, but don't regress B-02's tests.

## Test plan
- **manifest**: hash determinism/normalization; get/set round-trip; `set_fingerprint`
  preserves `region_hashes`.
- **heal/system (three-phase)**: llm-seeded fill stamps hash; human edit → re-heal
  leaves body byte-identical + idempotent; locked region + code move → reported
  healable=False; an UNLOCKED llm-seeded region still regenerates on code drift.
- **drift unit**: locked vs unlocked llm-seeded → healable False vs True.
- **B-02 retrofit**: human region advisory persists across a HASH heal until edited.
- Additive: all B-01/B-02 + prior tests green. ≥90% changed code; gate ≥90%.

## Dogfood
`manifest.py`/`heal.py`/`drift.py` are tracked. Reheal after green; `cdmon check`
+ `cdmon lint` exit 0. Pin the new manifest/heal signatures in ARCHITECTURE.md first.

## Out of scope
Full `llm`-mode backend *prose* authoring (a region with no renderer whose body
an LLM writes) — that's a separate slice (B-03b or folded into EPIC D retrieval);
B-03 treats llm-seeded as generated-until-locked, which is fully offline-testable.
The all-four-modes e2e is B-04.

## Constraints
K2/K5 (human-owned once edited; reported not silently changed), K7 (idempotent;
hashes stamped only on engine writes), K8, K10 (deterministic hash), K9.
