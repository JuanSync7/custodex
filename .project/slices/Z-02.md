# Slice Z-02 — remove redundant root `cdmon.yaml`; repoint to the dir layout

Final CONFIG-V2 slice. Removes the redundancy the dir-layout migration created
(cdx now has BOTH cdmon.yaml and config/cdmon/ describing the same surface) and
makes config/cdmon the canonical config for cdmon's own repo, while preserving the
single-file `load_config` path as a TESTED back-compat capability. See
`.project/spec/CONFIGV2.md` §7.

## Goal (validable)

cdmon's canonical self-config is `config/cdmon/` ONLY; the root `cdmon.yaml` is
removed. Default `cdx check` / `cdx coverage` (no `--config`) auto-detect the
dir layout and are CLEAN / meet the `--fail-under 95` gate. The single-file
`load_config` capability still works and is covered by tests using a SEPARATE
fixture (not cdmon's own config) and the `examples/` adopters. CI is updated to
the dir layout. Full suite green. Proven by: `cdmon.yaml` absent; default
`cdx check` exit 0 + `cdx coverage --fail-under 95` exit 0 via auto-detect;
single-file back-compat tests green; full `pytest` green; `cdx doctor` (or the
CI dogfood step) passes.

## In scope

- DELETE root `cdmon.yaml`. Confirm `_resolve_config`/auto-detect now selects
  `config/cdmon/index.yaml` by default from the repo root.
- Repoint everything that referenced cdmon.yaml (grep the WHOLE repo, not just
  tests: `.gitlab-ci.yml`, `README.md`, `pyproject.toml`, scripts, docs,
  `.project/`):
  - tests: any dogfood/system test that loaded `cdmon.yaml` → use the dir layout
    (default auto-detect) OR a separate single-file fixture. KEEP at least one
    explicit single-file back-compat test (own fixture under tmp_path, or via
    `examples/external-repo/cdmon.yaml`).
  - `.gitlab-ci.yml`: any `cdx ... --config cdmon.yaml` or step relying on the
    single file → dir layout (default). Keep the `--fail-under 95` gate.
  - README + module header comment in `config.py` (it mentions `cdmon.yaml`) →
    mention the dir layout as the default, single file as back-compat.
- Identify and remove any OTHER redundancy introduced by CONFIG-V2 (grep for dead
  code / now-unused branches). Be conservative: only remove what is genuinely
  unreachable or duplicated by the new design. Do NOT remove the single-file
  `load_config`/`CONFIG_TEMPLATE`/`cdx init` (single-file) path — it is the
  supported back-compat surface (keep + keep tested). Do NOT remove `examples/`
  (they cover the single-file path) unless they are exact duplicates of `demo/`
  (they are not — they are single-file adopters).
- Re-run the dogfood reheal if removing cdmon.yaml changes nothing in the docs (it
  shouldn't — config/cdmon already drives them) but VERIFY `cdx check` clean.
- Lint/type hygiene: the CONFIG-V2 work left a few pre-existing-style findings
  (e.g. a mypy call-arg on `docstyle`/`ConfigBundle.doc_style: object | None`,
  some ruff I001/SIM108/B905). If the CI gate runs ruff/mypy and these are NEW
  (introduced by CONFIG-V2, not baseline), fix them; if they are baseline, leave
  them. State which.

## Test plan
- `cdmon.yaml` no longer exists; `git status` shows it deleted.
- From repo root: `cdx check` (no --config) → exit 0 clean (auto-detect dir);
  `cdx coverage --fail-under 95` → exit 0 (100%); `cdx rpt` works.
- Single-file back-compat test green (separate fixture / examples).
- `examples/` tests + `test_dogfood` + full suite green
  (`pytest -q -m "not live_llm and not pg"`).
- If CI runs them: `ruff check` + `mypy custodex` show no NEW errors vs
  baseline.

## Out of scope
New features. (This is cleanup + repoint only.)

## Constraints
K7/K8/K10. Back-compat single-file path MUST remain working + tested. Do not
regress any behavior the suite asserts. Removal must be justified (genuinely
redundant), never of a tested capability.
