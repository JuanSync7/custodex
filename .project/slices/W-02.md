# Slice W-02 — canonical v2 templates + `cdmon init --v2` + `/config/templates` endpoint + Config page

Part of EPIC W. Two parts (backend then frontend). Gives ONE authoritative source
for the `config/cdmon/` format, a scaffolder adopters use, and a dashboard page
that displays the format. See `.project/spec/CONFIGV2.md` §1, §5.

## Goal (validable)

There is ONE canonical, tested set of `config/cdmon/` template strings (unit,
index, ignore, doc-style) that each ROUND-TRIP through their N-01..N-05 loaders.
`cdmon init --v2 [--config-dir config/cdmon] [--repo NAME]` scaffolds a complete,
loadable `config/cdmon/` from them. `GET /config/templates` serves them as JSON.
A dashboard Config page (`/config`) fetches and displays each template in a
labeled code block with a short explanation of the layout. Proven by: round-trip
tests (each template loads + validates); a `cdmon init --v2` test (scaffolds a dir
that `load_bundle` accepts); an endpoint test; and a Vitest test for the page.

## Part A — backend (`code_doc_monitor/`)

- New `code_doc_monitor/templates_v2.py`:
  `UNIT_TEMPLATE`, `INDEX_TEMPLATE`, `IGNORE_TEMPLATE`, `DOC_STYLE_TEMPLATE`
  (well-commented canonical examples matching CONFIGV2 §1.1–1.4, with the
  `../..` root convention). A `V2_TEMPLATES: dict[str,str]` aggregating them.
  `scaffold_config_dir(config_dir: Path, *, repo: str, now: str) -> None` writes
  index.yaml + one example unit + ignore.yaml + doc-style.yaml (+ the referenced
  writing templates already live in `templates/writing/`), substituting `repo`
  and `now`. The scaffolded dir MUST pass `load_bundle` (assert in a test).
- `cdmon init --v2 [--config-dir config/cdmon] [--repo NAME] [--force]` in cli.py:
  calls `scaffold_config_dir` (loud K8 if the dir exists and not `--force`).
  Keep the existing `cdmon init` single-file behavior unchanged (add `--v2` flag).
- Endpoint `GET /config/templates` in server/app.py: returns
  `{"unit":..., "index":..., "ignore":..., "doc_style":...}` (the V2_TEMPLATES,
  no auth — it's public reference). Deterministic.
- Tests: `tests/test_templates_v2.py` — each template parses via its loader
  (UNIT_TEMPLATE → load_unit_file after writing to a stem-matching file;
  INDEX_TEMPLATE → load_index_file; IGNORE → load_ignore_file; DOC_STYLE →
  load_doc_style with templates_root=templates/writing); `scaffold_config_dir`
  produces a `load_bundle`-valid dir; `cdmon init --v2` scaffolds + is loud on
  existing dir without --force. Endpoint test (TestClient): GET /config/templates
  returns all four. ≥90% branch coverage of new code; full suite green.

## Part B — frontend (`dashboard/src/`)

- `api/client.ts`: `configTemplates(): Promise<{unit,index,ignore,doc_style:string}>`
  → `GET .../config/templates` (no auth). Client test for the URL.
- `pages/Config.tsx`: global page (not per-repo). Fetches `configTemplates()` via
  useApi; renders a short intro paragraph on the `config/cdmon/` layout + four
  labeled `<pre><code>` blocks (Unit file, index.yaml, ignore.yaml,
  doc-style.yaml). Loading/error states. Match existing page conventions + CSS.
- `routing.ts` `linkToConfig()` = `/config`; `App.tsx` route `/config` →
  `<Config/>`; AppShell top nav: add a "Format" / "Config" link next to "API Docs".
- `index.css`: code-block styling reusing existing tokens (mono font already
  loaded).
- Tests: `pages/Config.test.tsx` (loading→ready; the four template bodies render)
  + the client test. `npm run test:run` AND `npm run build` pass.

## Out of scope
Sync button (W-03), standalone serve (L-01), demo content (M-01).

## Constraints
K7/K10 (templates deterministic; init idempotent under --force semantics — loud
without it), K8 (loud), K0 (no new deps). Existing single-file `cdmon init`
unchanged. TS strict build passes; match dashboard conventions.
