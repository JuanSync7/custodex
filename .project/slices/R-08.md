# Slice R-08 — `cdmon wiki` CLI + traceability CI gate (EPIC R close-out)

EPIC R, phase R7. The single regeneration entry point: `cdmon wiki` renders every
EPIC-R derived artifact from its source, `--check` proves freshness, and the
now-complete traceability matrix becomes a CI gate. Closes EPIC R.

## Goal (validable)
1. **One command, all wikis.** `cdmon wiki` writes the four canonical artifacts
   from their single sources (catalog yaml, tests' docstrings, source AST):
   `feature-doc/FEATURES.md`, `feature-doc/wiki/TEST_WIKI.md`,
   `feature-doc/wiki/SOURCE_WIKI.md`, `feature-doc/wiki/TRACEABILITY.md`. A shared
   `WIKI_TARGETS` (path → render thunk) is the single source of the output set.
2. **Idempotent (K7).** Two `cdmon wiki` runs leave the tree byte-identical; the
   second reports all-unchanged.
3. **Freshness gate (K8).** `cdmon wiki --check` exits 0 when all four are fresh,
   nonzero listing every stale file when not — without writing.
4. **Traceability gate.** `cdmon trace --fail-on-gap` exits 0 on the real tree (the
   matrix is complete after R-04/R-06); both `cdmon wiki --check` and
   `cdmon trace --fail-on-gap` are added to `.gitlab-ci.yml` (offline, K4).
5. **Deterministic (K10).** All four renders are pure functions of their sources.

## Design
See ARCHITECTURE.md `cdmon wiki` pin. `cli.py` adds `wiki(check: bool = False)`.
A module-level `WIKI_TARGETS: dict[Path, Callable[[], str]]` (or a small helper in
a new `wiki.py` if cli.py grows too large — prefer a thin `code_doc_monitor/wiki.py`
holding `WIKI_TARGETS` + `regenerate(write: bool) -> list[(path, changed)]` so the
logic is unit-testable without the CLI). The CLI command is a thin wrapper. Reuses
featurecatalog/testwiki/srcindex/traceability renderers. Loud (K8) on a render
error. No new dep (K0).

## Test plan
- **unit (tests/unit/test_wiki.py):** `regenerate(write=True)` writes all four
  targets; a second call reports all-unchanged (idempotent K7); `regenerate(write=False)`
  (check) returns stale=[] on a fresh tree and the right stale list after a target is
  mutated; `WIKI_TARGETS` covers exactly the four canonical paths; renders are
  deterministic. Tag `Features: FEAT-REFERENCE-…` (the wiki regen is a reference feature
  — ADD a `FEAT-REFERENCE-007` "cdmon wiki regeneration + freshness gate" to
  reference.yaml, with a demo case + this test tagging it, keeping the matrix complete).
- **system (tests/system/test_wiki_cli.py):** `cdmon wiki` then `cdmon wiki --check`
  exits 0 (idempotent through the CLI); after touching a wiki, `--check` exits nonzero;
  `cdmon trace --fail-on-gap` exits 0 on the real tree.
- Full gate green; `cdmon wiki` (regenerate) leaves the committed wikis fresh;
  `cdmon check`/`lint`/`coverage --fail-under 95` exit 0; new `wiki.py` added to the
  coverage waiver.

## Dogfood + CI
- Run `cdmon wiki` so the four artifacts on disk match HEAD; `cdmon wiki --check`
  then exits 0.
- Add to `.gitlab-ci.yml` (in/near the offline test job, or a new `docs:wiki` job):
  `cdmon wiki --check` and `cdmon trace --fail-on-gap`. Offline (K4).

## Out of scope
A Confluence/Atlassian export (a later follow-on — the in-repo markdown wikis are
the deliverable); HTML twins of the wikis; per-test prose enrichment beyond what
already exists.

## Constraints
K7 (idempotent regen), K8 (`--check`/`--fail-on-gap` loud nonzero on staleness/gap),
K10 (deterministic renders), K4 (CI steps offline — no network), K0 (no new dep).
This slice makes EPIC R self-sustaining: the reference can't silently drift from the
code, demos, or tests again.
