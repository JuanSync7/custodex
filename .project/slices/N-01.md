# Slice N-01 — config-v2 models + `config/cdmon/` dir loader + merge

Part of EPIC N (CONFIG-V2). First slice. Adds the multi-file `config/cdmon/`
layout as a *projection* onto the existing `MonitorConfig`. No downstream module
(drift/coverage/heal/manifest/server) changes. See `.project/spec/CONFIGV2.md`
§0–§1.2.

## Goal (validable)

Given a `config/cdmon/` directory with an `index.yaml` (globals + units list) and
N unit `*.yaml` files (each with a `---` frontmatter block, `dir-covered`,
`source-files-format`, `documents`), `load_config_dir(dir)` returns ONE
`MonitorConfig` equal to what the merged single-file config would produce, with
documents concatenated in index order. Frontmatter and structural rules are
enforced loudly (typed `ConfigError`). Proven by automated tests building a real
fixture `config/cdmon/` tree and asserting the merged `MonitorConfig` equals a
hand-written expected one, plus loud-failure tests.

## In scope — `code_doc_monitor/config.py` (extend; pure, K0/K8/K10)

Add models (frozen, `extra="forbid"`):

```python
class UnitFrontmatter(BaseModel):
    cdmon_config_version: str       # alias "cdmon-config-version"; must == "2.0.0"
    unit: str
    title: str
    owner: str
    created: str
    updated: str

class UnitFile(BaseModel):
    frontmatter: UnitFrontmatter
    dir_covered: tuple[str, ...]            # alias "dir-covered"; >=1
    source_files_format: tuple[str, ...]    # alias "source-files-format"; >=1, each starts "."
    documents: tuple[DocumentSpec, ...]     # >=1

class IndexFrontmatter(BaseModel):
    cdmon_config_version: str               # must == "2.0.0"
    repo: str
    generated_by: str                       # alias "generated-by"
    updated: str

class IndexUnitRef(BaseModel):
    file: str

class IndexFile(BaseModel):
    frontmatter: IndexFrontmatter
    root: str = ".."                         # repo root relative to config/cdmon/
    version: str = "2.0.0"
    apply_default: bool = False
    backend: BackendConfig = BackendConfig()
    agent: AgentConfig = AgentConfig()
    central: CentralConfig = CentralConfig()
    region_templates: dict[str, RegionTemplate] = {}
    coverage: CoverageConfig = CoverageConfig()
    units: tuple[IndexUnitRef, ...]
    ignore: str = "ignore.yaml"
    doc_style: str = "doc-style.yaml"        # alias "doc-style"
```

Use pydantic field aliases (`populate_by_name=True`) so YAML keys are
hyphenated (`dir-covered`) while Python attrs are snake_case. Parse the `---`
frontmatter block with the SAME splitter style as `manifest.parse_doc` (a YAML
front-matter fence) — factor a tiny `_split_frontmatter(text) -> (meta, body)`
helper here (do not import manifest; avoid cycles).

Functions:

```python
def load_unit_file(path: Path) -> UnitFile: ...          # frontmatter+body, validates unit==stem
def load_index_file(path: Path) -> IndexFile: ...
def load_config_dir(config_dir: Path) -> MonitorConfig: ...
```

`load_config_dir` algorithm:
1. Read `index.yaml` (loud if absent). Parse frontmatter + body → `IndexFile`.
2. For each `units[].file` in order: `load_unit_file(config_dir/file)`. Loud if a
   listed file is missing. (Cross-checking the *reverse* — on-disk units not in
   index — is N-02; keep N-01 index-driven.)
3. Validate: `unit == file stem`; `cdmon-config-version == "2.0.0"`;
   `source-files-format` entries each start with `.`; `dir-covered` non-empty;
   duplicate document `id` across all units → loud; `dir-covered` overlap across
   units → loud (a path under two units' dir-covered).
4. Build `MonitorConfig(version=..., root=index.root, documents=<concat>,
   region_templates=index.region_templates, backend=..., agent=..., central=...,
   apply_default=..., coverage=index.coverage)`.

Expose the per-unit scoping (`dir_covered`, `source_files_format` keyed by unit,
and which unit each document came from) via a NEW return seam so N-03/N-04 can
use it WITHOUT re-reading disk. Add a light dataclass/model:

```python
class ConfigBundle(BaseModel):     # frozen
    config: MonitorConfig
    index: IndexFile
    units: tuple[UnitFile, ...]
    config_dir: str
    def unit_for_document(self, doc_id: str) -> UnitFile | None: ...
def load_bundle(config_dir: Path) -> ConfigBundle: ...   # load_config_dir = load_bundle(...).config
```

CLI auto-detect (`code_doc_monitor/cli.py`): add a resolver
`_resolve_config(config: Path) -> tuple[MonitorConfig, Path]` used by `_load`:
if `config` is a directory OR `config/cdmon/index.yaml` exists relative to cwd,
use `load_config_dir`; else `load_config(config)` as today. Keep the default
`--config cdmon.yaml`. Do NOT break any existing CLI behavior or test.

Add to `config.py` `__all__`. Update `ARCHITECTURE.md` with the new contracts.

## Test plan (`tests/test_config_v2.py`, new)

- **unit**: frontmatter splitter (well-formed, missing fence, malformed YAML →
  ConfigError); UnitFile alias round-trip; version!="2.0.0" loud; bad extension
  (no leading dot) loud; unit!=stem loud; empty dir-covered loud.
- **integration / real fixture**: build a `config/cdmon/` under tmp_path with an
  index + 2 units (mirroring the foundation + agent-workflow shapes from
  cdmon.yaml), assert merged `MonitorConfig.documents` equals expected tuple
  (ids, paths, audiences, code_refs) in index order; assert globals (backend,
  region_templates, coverage.waive) come from index.
- **loud**: unit listed in index but file missing; duplicate doc id across units;
  dir-covered overlap across units.
- **bundle**: `load_bundle(...).unit_for_document("agent-workflow")` returns the
  right unit; `unit.source_files_format == (".py",)`.
- **CLI back-compat**: existing single-file `cdmon check`/`load_config` path
  unchanged (add a test that a single-file config still loads via `_resolve_config`).
- Coverage of new code in `config.py` ≥ 90% (whole-pkg gate ≥ 90%, branch).
