# Slice G-01/G-02 ŌĆö `cdmon init --central` + `cdmon doctor`

First EPIC-G slices. Makes cdmon droppable into any repo: a one-command bootstrap
that wires the central reporting + a preflight check.

## Goal (validable)
(G-01) `cdmon init --central <url> [--repo-id ID] [--token-env VAR]` writes a
`cdmon.yaml` whose `central:` block is configured for HTTP reporting (sink=http,
url, repo_id, auth_env) ŌĆö round-trips through `load_config` and is ready to
`cdmon register` + report; without `--central` it writes today's offline template
(unchanged). (G-02) `cdmon doctor [--config]` checks prerequisites (config loads;
backend reachable/configured; central url+token present when sink=http; optional
extras present for the chosen backend/agent) and prints a clear PASS/WARN/FAIL
report, exit 0 if no FAIL. Proven by: init-writes-central-config round-trip test,
init-without-central unchanged, doctor PASS on a good config + FAIL on a broken one.

## In scope ŌĆö `cli.py` (+ a small `doctor.py` if cohesive)
### G-01 ŌĆö `cdmon init --central`
- Extend the existing `init` command: `--central URL`, `--repo-id ID` (default:
  derive from cwd dir name or require it), `--token-env VAR` (default
  `CDMON_GITLAB_TOKEN`-style, document), optional `--repo-url`.
- When `--central` given: emit a config whose `central:` = `{sink: http, url,
  repo_id, repo_name?, repo_url?, auth_env, outbox}`. Must round-trip through
  `load_config` and satisfy `make_sink`'s http requirements (repo_id present).
- Without `--central`: byte-identical to today's template (existing init tests green).
- Optionally also emit a `.env.example` / a note about `VITE_API_BASE` for the
  dashboard ŌĆö keep minimal, document.

### G-02 ŌĆö `cdmon doctor`
- `RepoDoctor`/`run_checks(config, config_dir) -> list[Check]` (pure-ish; a `Check`
  = name/status(PASS|WARN|FAIL)/detail). Checks: config loads; documents resolve
  (paths exist or are MISSING_DOC-creatable); backend kind valid + its prereq
  (claude CLI / `$ANTHROPIC_API_KEY` / langgraph extra) ŌĆö WARN not FAIL when a
  prereq is merely absent in this env; central: when sink=http, url + the
  `auth_env` token presence (WARN if token env unset); the `[server]`/`[agent]`
  extras importability (WARN if absent). Side-effect-free except reading env.
- `cdmon doctor` prints the checks, exit 0 unless any FAIL.
- Keep network OUT (don't actually hit the central url ŌĆö that's a connectivity
  check we only do if a `--ping` flag is given AND a transport is injected; default
  doctor is offline). Document.

## Test plan (`tests/test_cli.py` + `tests/test_doctor.py`)
- init --central writes a config that `load_config` accepts + `make_sink` builds an
  HttpSink from (repo_id present); init without --central unchanged (existing tests).
- doctor: a good offline config ŌåÆ all PASS/WARN, exit 0; a config with sink=http
  but no url ŌåÆ FAIL, exit 1; backend prereq absent ŌåÆ WARN (exit 0); deterministic
  check ordering.
- No network; injected env for token-presence checks.
- Additive: all prior tests green. Ōēź90% new code; gate Ōēź90%.
