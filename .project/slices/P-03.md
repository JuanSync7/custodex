# Slice P-03 ‚Äî Additional extractors via the protocol (registry-routed symbols)

EPIC P. Makes symbol extraction resolve through the P-01 `Extractor` registry by
the ref's language instead of a hardcoded Python path, so **a new language is a
`register_extractor()` call, never an edit to the engine's control flow** (K0).
No new heavy parser ships in this slice ‚Äî the goal is the extensibility *contract*,
proven by a test-registered stub extractor (the MockBackend pattern).

## Goal (validable)
1. **Registry-routed extraction.** `_symbols_for_ref` resolves
   `get_extractor(_symbol_language(ref)).extract(path)` ‚Äî the hardcoded
   `extract_file` (Python) path is gone. `_symbol_language(ref)` = `ref.lang` when
   explicit, else the suffix map (`auto`), defaulting to `"python"` (the pre-P3
   behaviour for every symbol ref).
2. **A new language is pure registration (K0).** A test registers a non-Python
   stub extractor (`register_extractor(stub, suffixes=(".toy",))`) and
   `build_document_surface` returns the stub's symbols for both an explicit
   `lang: toy` ref AND an `auto` ref resolved from the `.toy` suffix ‚Äî with ZERO
   engine edit. An unregistered language ‚Üí `ExtractionError` (K8). Python `.py`
   refs with `lang: auto` are byte-unchanged (the golden composites still hold).

## Design
- **extract.py.**
  - `_SYMBOL_LANG_BY_SUFFIX: dict[str, str] = {".py": "python"}` ‚Äî the only built-in
    entry; the engine holds no other language list (K0).
  - `register_extractor(extractor, *, suffixes=())` now also maps each `suffix ‚Üí
    extractor.language`, so a `symbols` ref with `lang: auto` resolves the new
    extractor by file extension. Registering a language is the sole step to support
    it.
  - `_symbol_language(ref) -> str`: explicit `ref.lang` wins; `auto` ‚Üí
    `_SYMBOL_LANG_BY_SUFFIX.get(suffix, "python")`. Resolved via `get_extractor`, so
    an unknown language is loud (K8).
  - `_symbols_for_ref` calls `get_extractor(_symbol_language(ref)).extract(...)`.
    `extract_file` stays a Python wrapper for existing callers (`inventory`, etc.).
- **config.py.** `CodeRef.lang` opened from a closed `Literal[...]` to `str`
  (default `auto`) so a ref can name any registered language. **Back-compat
  loosening:** every previously-valid value still validates; the only behavioural
  change is that an unknown language is now caught at extraction (K8) rather than
  config-load. The built-in switch/record parsers still recognize only
  python/shell/tcl/json.

## Schema / migration (K6)
No `ReviewRecord`/surface wire change. `CodeRef.lang` loosens its TYPE (Literal ‚Üí
str) ‚Äî a superset, so all existing configs load unchanged; this is an additive
relaxation, not a breaking change. ARCHITECTURE.md pins the `register_extractor`
suffix param, `_symbol_language`, and the registry-routed `_symbols_for_ref`.

## Test plan
- **unit (test_extract.py):** routes by explicit `lang`; `auto` infers a registered
  suffix; `register_extractor` maps the suffix; unknown language ‚Üí `ExtractionError`
  (K8); a `.py` `auto` ref still uses Python (back-compat). The stub extractor
  satisfies the `Extractor` Protocol and never imports the target (K0). Registry +
  suffix-map mutations are restored in `finally` so the global state is clean.
- Additive: all prior tests green (golden composites unchanged); gate ‚â•90%
  (`--cov-branch`).

## Dogfood
Opening `CodeRef.lang` + the new public surface in `extract.py`
(`_SYMBOL_LANG_BY_SUFFIX`, the widened `register_extractor`, `_symbol_language`)
drift the eng-guide API docs on the signature tier; rehealed via
`cdx monitor --apply --config config/cdmon`; `cdx check` + `cdx lint` exit 0.

## Out of scope
A REAL non-Python parser (e.g. tree-sitter/regex extractor for Rust/Go) ‚Äî that is a
follow-on registration that needs an optional dependency (K0 heavy-deps-are-extras)
and does not change the engine. Region anchors (P-04).

## Constraints
K0 (no target knowledge; a new language is a registration, not an engine edit;
AST/parse-only, no import of target), K6 (additive type loosening; configs
unchanged), K8 (loud on unknown language at extraction), K9 (TDD, additive), K10
(deterministic; golden composites unchanged).
