# Slice C-01/C-02 — `cdmon sync-pr` (doc patch producer) + CI check-gate

First EPIC-C slices. Produces the **docs-PR content** (a patch of what healing the
docs changes) and adds a fast offline CI gate. The bot that actually opens the MR
is C-03; this slice is the git-free, fully-testable core.

## Goal (validable)
On a fixture repo with drift, `cdmon sync-pr` heals the docs and emits a unified
diff (patch) of exactly the changed doc files + a summary, exit 0; `--dry-run`
emits the SAME patch but leaves the working tree byte-unchanged; a clean repo (or
a second run after apply) emits an EMPTY patch (idempotent, K7). The CI gate job
runs `cdmon check` + `cdmon lint` offline and fails on drift. Proven by:
sync-pr-with-drift (patch non-empty, files healed), `--dry-run` (same patch, files
unchanged), clean/second-run (empty patch), and a `.gitlab-ci.yml` lint/parse check.

## In scope
### `code_doc_monitor/syncpr.py` (new) — git-free, testable
```python
class SyncResult(BaseModel):           # frozen, extra="forbid"
    patch: str                         # concatenated per-file unified diffs ("" if none)
    changed_paths: tuple[str, ...]     # repo-relative doc paths that changed
    summary: str                       # human one-liner ("2 doc(s) updated" / "clean")

def sync_pr(monitor: Monitor, *, dry_run: bool = False) -> SyncResult: ...
```
- Snapshot each document's current text (or None if missing) BEFORE.
- Run `monitor.run(apply=True)` (heals in place via the existing pipeline — backend
  verdicts + heal, so it honors B-02/B-03 region authority automatically: human
  regions won't appear in the patch).
- Read each document AFTER; for changed ones, build a unified diff
  (`difflib.unified_diff`, deterministic, `fromfile=a/<path>` `tofile=b/<path>`).
- If `dry_run`: restore every doc to its before-text (and DELETE any file the run
  created that didn't exist before — e.g. a MISSING_DOC stub) so the tree is
  byte-identical to the start. Verify-restore is the dry-run contract.
- Pure of network/LLM by default (mock backend); deterministic patch (K10).

### `cli.py` — `cdmon sync-pr [--config] [--out FILE] [--dry-run]`
- Default: apply + print patch (or write to `--out`), print summary to stderr,
  exit 0. `--dry-run`: compute + print patch, no mutation. K8 clean errors.

### `.gitlab-ci.yml` — the C-01 gate
- Add a fast `docs:gate` job (offline, no LLM) running `cdmon check --config
  cdmon.yaml` and `cdmon lint`, failing the pipeline on drift. Runs on MRs +
  branches. Keep it separate from `tests:offline` (two orthogonal signals).

## Test plan (`tests/test_syncpr.py` + `tests/test_cli.py`)
- drift → `sync_pr()`: patch contains the doc path + the healed lines; the doc
  file on disk is now healed; `changed_paths` exact.
- `--dry-run`: identical patch, but every doc byte-identical to before (and no
  stray stub left); assert with a content snapshot.
- clean repo → empty patch, summary "clean"; idempotent (apply then re-run → empty).
- human/llm-seeded region in the fixture: its body is NOT in the patch (authority
  respected — reuses B-02/B-03, so this is a regression guard).
- CLI: `--out` writes the patch file; default prints it; exit 0; bad config → clean error.
- `.gitlab-ci.yml` stays valid YAML (a test can `yaml.safe_load` it).
- Additive: all prior tests green. ≥90% new code; gate ≥90%.

## Dogfood
`syncpr.py`/`cli.py` are not dogfood-tracked → no reheal expected; confirm `cdmon

## Out of scope

note the intent in a comment).
