# Slice B-01 ŌĆö region authority mode (schema)

First slice of EPIC B. Declares per-region **authority** so a human can own
sections alongside generated/LLM ones. Schema + validation only ŌĆö behavior wiring
is B-02 (human), B-03 (llm-seeded), B-04 (mixed e2e). No drift/heal change yet.

## The four modes (define in code + docs)
- `generated` (DEFAULT) ŌĆö pure projection of code (symbol table / index /
  fingerprint). Heal overwrites mechanically, no LLM. (Today's behavior.)
- `llm` ŌĆö prose authored by the backend; on code drift the backend re-proposes;
  heal applies the LLM fix, not a mechanical render.
- `human` ŌĆö a human owns it; heal NEVER writes it; code drift only raises an
  advisory signal "the code this section describes changed ŌĆö review".
- `llm-seeded` ŌĆö behaves as `llm` until a human edits it (detected by a stored
  per-region content hash diverging), then locks to `human`.

## Goal (validable)
A `cdmon.yaml` whose a doc declares `region_modes: {symbols: generated, intro:
human}` round-trips through `load_config` to the exact enum values; an unspecified
region defaults to `generated` (old configs without `region_modes` load
unchanged ŌĆö K6 additive); an unknown mode string ŌåÆ `ConfigError`; a `region_modes`
key naming a region **not** in that doc's `region_keys` ŌåÆ `ConfigError` (K8).
Proven by round-trip + both loud-error tests.

## In scope ŌĆö `config.py` only

```python
class RegionMode(str, Enum):

    LLM = "llm"
    HUMAN = "human"
    LLM_SEEDED = "llm-seeded"

class DocumentSpec(BaseModel):
    ...
    region_modes: dict[str, RegionMode] = {}   # region id -> mode; absent => generated
```
- Add a pydantic `model_validator` on `DocumentSpec`: every key of
  `region_modes` must be in `region_keys`, else raise (surfaced as `ConfigError`
  by `load_config`'s existing wrapping). Keep models frozen/extra="forbid".
- Provide a helper `DocumentSpec.mode_for(region_id) -> RegionMode` returning the
  declared mode or `RegionMode.GENERATED`. (B-02/03/04 consume this ŌĆö one accessor,
  no scattered `.get(..., GENERATED)`.)
- Update `CONFIG_TEMPLATE` with a commented `region_modes:` example that still
  round-trips (existing template test must stay green).
