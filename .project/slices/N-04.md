# Slice N-04 — `cdmon rpt` → `config/cdmon/coverage.rpt`

Part of EPIC N (CONFIG-V2). Builds on N-01/N-02/N-03. See
`.project/spec/CONFIGV2.md` §3.

## Goal (validable)

`cdmon rpt [--write]` computes a coverage report from the dir-layout config and
emits it as YAML-with-frontmatter `config/cdmon/coverage.rpt`. It contains: a
`summary` (scanned/documented/waived/ignored/uncovered file counts + percent), a
per-`units` breakdown, and an `undocumented` list where each entry names the file
AND the `suggested_unit` it should be declared in (the unit whose `dir-covered`
contains it and whose `source-files-format` includes its extension; else `null`
with a reason). Without `--write` it prints to stdout (read-only, K1). With
`--write` it writes the file idempotently (K7: re-run with no code/config change
→ byte-identical file). Proven by a fixture repo + golden assertion on the parsed
report structure, an idempotency test, and a `suggested_unit` test.

## In scope

New module `code_doc_monitor/report.py` (pure; depends on config + coverage +
inventory only):

```python
class CoverageRpt(BaseModel):           # frozen — the parsed report shape
    cdmon_report_version: str           # "1.0.0"
    kind: str                           # "coverage"
    repo: str
    ref: str | None
    summary: RptSummary
    units: tuple[RptUnit, ...]
    undocumented: tuple[RptUndocumented, ...]

def build_coverage_rpt(bundle: ConfigBundle, repo_root: Path, *, ref: str | None) -> CoverageRpt: ...
def render_rpt(rpt: CoverageRpt) -> str: ...   # YAML body + leading --- frontmatter --- block
def write_rpt(config_dir: Path, text: str) -> None: ...   # loud K8
```

- Reuse `effective_coverage` + the real `discover_files`/`discover_symbols`/
  `resolve_coverage` to get the universe. Counts:
  - `scanned_files` = files in the universe (format-matched, under some unit
    dir-covered, minus ignore). `documented_files`/`waived_files`/`uncovered_files`
    from the `CoverageReport` properties. `ignored_files` = count of repo files
    that matched the ignore/exclude set within unit dirs (informational —
    document precisely how it's counted; keep deterministic).
  - `percent` = `CoverageReport.percent_files` (waived removed from both sides);
    format to 2 decimals deterministically.
- Per-unit breakdown: for each unit attribute each universe file to the unit
  whose `dir-covered` contains it; `scanned/documented/percent/uncovered` per unit.
- `undocumented`: each uncovered (undocumented, non-waived) file → its
  `suggested_unit` (filename of the unit by dir-covered + format match) + reason;
  if no unit matches, `suggested_unit: null` + a reason.
- `render_rpt`: emit a `---` frontmatter block (report-version/kind/repo/ref/
  generated-by) then the YAML body. Deterministic key order; sorted lists. The
  output MUST round-trip: parsing the rendered text back yields an equal
  `CoverageRpt` (add a test). Use the project's injected clock seam only if a
  timestamp is included — PREFER `ref` (branch/commit) over a wall-clock in the
  file so re-runs are byte-stable (K7/K10); if you include `generated` time, it
  breaks idempotency — DO NOT include wall-clock in the written file.

`code_doc_monitor/cli.py`: add `cdmon rpt [--config-dir config/cdmon] [--write]
[--ref REF]`. Default prints; `--write` writes `coverage.rpt`. Resolve `ref`
from `--ref` else leave null (a later sync slice fills it). Exit 0 normally; K8
loud on config errors.

## Test plan (`tests/test_report.py`, new)

- **unit**: percent formatting (2 dp, 100.0, 0.0, n/a when denominator 0);
  suggested_unit resolution (matching unit, format mismatch → different/none,
  no-unit → null+reason); per-unit attribution.
- **integration (golden)**: fixture repo + config/cdmon with 2 units, one fully
  documented + one with an undocumented `.py` and a `.log` (ignored by format) →
  assert parsed CoverageRpt summary counts, per-unit percents, and the
  undocumented entry's suggested_unit.
- **round-trip**: `parse(render_rpt(r)) == r`.
- **idempotency (K7)**: `cdmon rpt --write` twice → identical bytes; no wall-clock
  in the file.
- **cli**: default prints (no file written, K1); `--write` creates the file;
  `--ref main` lands in frontmatter.
- ≥90% branch coverage of new code; full suite green.
