# Slice R-02 — Populate the golden feature catalog (all subsystems)

EPIC R, phase R1 (population). R-01 shipped the loader + schema + a 6-feature
`extract` seed. R-02 fills `feature-doc/catalog/` with the COMPLETE, source-verified
catalog of every cdx feature, grouped one yaml file per subsystem.

## Goal (validable)
1. **Complete coverage.** Every subsystem has a `feature-doc/catalog/<subsystem>.yaml`
   enumerating its distinct, user-meaningful features.
2. **Correct.** `load_catalog(feature-doc/catalog, known_modules=<real module set>)`
   loads with ZERO errors — i.e. no duplicate ids, every `modules` entry is a real
   `custodex` module, every id well-formed. Every headline public symbol
   named in a summary actually exists (probed).
3. **Rendered.** `feature-doc/FEATURES.md` regenerated from the catalog.

## Method (orchestrated fan-out)
One subagent per subsystem cluster, each: reads its real source module(s) +
ARCHITECTURE/SPEC, enumerates distinct features (NOT one-per-function), VERIFIES
every named symbol via `hasattr` import probes, and writes ONLY its own
`catalog/<subsystem>.yaml`. The `load_catalog` loader is the cross-file
consistency gate (unique ids, real modules, schema); the orchestrator runs the
authoritative whole-dir load + a consolidated headline-symbol probe + a manual
spot-check, then renders `FEATURES.md`.

## Result
186 features across 18 subsystems: agent 8, backends 8, cli 22, config 12,
configv2 15, coverage 10, drift 10, extract 6, heal 9, layout 9, learn 6,
manifest 9, monitor 9, pr 11, quality 9, record 13, reference 2, server 18.
Whole-catalog load clean under the real module set; headline symbols all probed
real (one apparent miss — `render_markdown` — was a probe-key error: the feature
correctly attributes it to `[build]`, where it lives).

## ID-prefix ⇄ subsystem normalization
The `server` cluster initially split two client-side `registry.py` features under
a `subsystem: registry` while keeping `FEAT-SERVER-*` ids; normalized to
`subsystem: server` so id-prefix and subsystem agree (the central-reporting story
spans client + server).

## Out of scope
Traceability matrix (R-03); wiring `demos`/`tests` slots (R3/R5); `cdx wiki`
regeneration command (R-08); per-feature module-completeness assertion (an
automated "no orphan public capability" check — a follow-on once the source index
R-06 exists).

## Constraints
K0 (no new dep), K8 (loader loud on every malformed/duplicate/unknown-module
input), K10 (deterministic sorted load + render). The catalog is hand-maintained
data; correctness is upheld by the loader + symbol probes, not by trust.
