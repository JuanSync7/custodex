# Slice E-02 — repo registration client

EPIC E. A repo announces itself to the central server before/while reporting:
sends its identity (+ a one-time token to mint, or its bearer for auth). Mirrors
E-01's injected-transport, never-network-in-tests discipline. CLIENT-SIDE only.

## Goal (validable)
`register_repo(identity, *, transport, dry_run=False)` builds a deterministic
registration payload (`RepoIdentity` + optional display fields) and submits it via
an INJECTED transport, returning the server response; `--dry-run` returns the
payload without calling; a `cdmon register [--config]` command runs it from config.
Proven by: payload-shape test (fake transport captures it), dry-run no-call, and a
CLI test — NO network.

## Test plan (`tests/test_registry.py` + `tests/test_cli.py`)
- payload shape: `register_repo` with a fake transport → captured `RegistrationPayload`
  has the right repo_id/identity + schema_version.
- dry-run: transport NOT called; payload returned.
- missing url/repo_id → loud error (K8).
- default transport: cover construction + missing-env loud path; real urlopen the
  only `# pragma: no cover` leaf (stub to cover lazy build), exactly like E-01.
- CLI `register` + `--dry-run`.
- Additive: all prior tests green. ≥90% new code; gate ≥90%.
