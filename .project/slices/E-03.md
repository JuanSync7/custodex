# Slice E-03 — central server skeleton (FastAPI, `[server]` extra)

EPIC E. The central service: a FastAPI app that ingests repo registrations +
review records over the SHARED schemas (no DTOs, K6). This slice uses an
**in-memory store behind a `Store` Protocol** — E-04 swaps in SQLAlchemy/Postgres
without touching the routes. Tested entirely with FastAPI `TestClient` (no socket).

## Goal (validable)
`POST /repos` (a `RegistrationPayload`) registers a repo; `POST /ingest` (an
`IngestEnvelope`) stores a record under its repo; `GET /repos` lists registered
repos; `GET /repos/{repo_id}/records` returns that repo's records; an unknown
`repo_id` on ingest → 404 (or auto-registers — decide+document); a malformed body
→ 422 (FastAPI/pydantic validation against the shared schema). Proven by
TestClient tests: register→ingest→read round-trip, validation rejection, unknown-repo
handling. NO real socket/DB.
