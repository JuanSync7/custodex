# Slice Z-01b — migrate cdmon's own config to `config/cdmon/` + reheal

Part of EPIC Z. Dogfoods the dir layout on cdx itself, using nested units
(Z-01a). See `.project/spec/CONFIGV2.md` §7.

## Goal (validable)

cdx monitors ITSELF via `config/cdmon/` (nested units: `core` =
`custodex`, `agent` = `custodex/agent`, `server` =
`custodex/server`), mirroring exactly the 12 documents + waivers +
region_templates of the current root `cdmon.yaml`. `cdx check` (auto-detecting
the dir layout) is CLEAN, `cdx coverage` meets the existing CI `--fail-under`
gate (same % as before, ALL current modules documented), and the full test suite
stays green. Proven by: `cdx check` exit 0 via the dir layout, `cdx coverage
--fail-under <gate>` exit 0, `load_bundle(config/cdmon).config.documents` has the
same 12 ids, and `pytest` green.

## In scope

- Create `config/cdmon/` for THIS repo, mirroring root `cdmon.yaml`:
  - `index.yaml`: frontmatter (repo `custodex`), `root: "../.."`, backend
    mock, central none, `region_templates` (the `api-index` template),
    `coverage.waive` for the three `__init__.py` aggregators (exact same reasons),
    `units: [core.yaml, agent.yaml, server.yaml]`, ignore + doc-style pointers.
  - `core.yaml`: dir-covered `[custodex]`, source-files-format `[".py"]`,
    documents = the 10 root-level docs (`api-index`, `foundation`, `pipeline`,
    `remediation`, `coverage-system`, `layout-build`, `pr-loop`, `central-client`,
    `learning`, `ops`) with their EXACT current code_refs (copy from cdmon.yaml,
    including the shared `schema.py` symbol selections + any modules prior slices
    added: docstyle.py, templates_v2.py, configsync.py, report.py, …).
  - `agent.yaml`: dir-covered `[custodex/agent]`, format `[".py"]`,
    documents = `[agent-workflow]` (the 5 agent modules).
  - `server.yaml`: dir-covered `[custodex/server]`, format `[".py"]`,
    documents = `[server]` (app/store/db + standalone.py if it now exists).
  - `ignore.yaml`: `gitignore: true` + patterns mirroring cdmon.yaml's coverage
    exclude (`**/.*/**`, `**/__pycache__/**`, `**/.venv/**`).
  - `doc-style.yaml`: defaults (eng-guide → api-reference/precise/reference-dense/
    engine-domain) + any per-doc overrides; references the existing
    `templates/writing/` (repo_root templates_root).
  - `coverage.rpt`: generated via `cdx rpt --write`.
- **Ensure NO module is undocumented** (the self-coverage gate): run `cdx
  coverage` against the dir layout; any new module added across this whole effort
  that is not yet in a document (e.g. `report.py`, `configsync.py`,
  `server/standalone.py`, `templates_v2.py`, `docstyle.py`) must be added to the
  appropriate document's `code_refs` AND have its doc region rehealed
  (`cdx monitor --apply`) so coverage meets the gate. Mirror these additions in
  the OLD `cdmon.yaml` too (keep both consistent until Z-02 removes cdmon.yaml).
- Keep root `cdmon.yaml` IN PLACE this slice (back-compat; Z-02 removes it). Both
  configs must describe the same surface so the shared docs stay in sync under
  either.
- Update any test that explicitly loads `cdmon.yaml` for dogfood so it ALSO passes
  under the dir layout (find them: grep tests for `cdmon.yaml`). If `test_dogfood`
  asserts a specific config path, make it tolerant of/aligned with the dir layout
  (prefer pointing dogfood at the dir layout; keep a single-file back-compat test
  using a SEPARATE fixture, not cdmon's own config).

## Test plan
- `cdx check` (dir layout, from repo root) → exit 0 clean.
- `cdx coverage --fail-under <the CI gate value>` → exit 0 (find the gate in
  `.gitlab-ci.yml`/`pyproject.toml`).
- `load_bundle(Path("config/cdmon")).config.documents` → same 12 ids as
  `load_config(Path("cdmon.yaml")).documents`.
- `cdx rpt --write` twice → byte-identical `config/cdmon/coverage.rpt`.
- Full suite `pytest -q -m "not live_llm and not pg"` → green (incl. test_dogfood).
- Add a test asserting cdmon's dir layout loads + the document id set equals the
  single-file config's (parity), so future drift between them is caught.

## Out of scope
Removing cdmon.yaml + redundant scaffolding (Z-02).

## Constraints
K7 (reheal idempotent; rpt idempotent), K8, K10. The dir layout must produce the
SAME drift/coverage verdicts as the single file (parity is the whole point of
dogfooding).
