# A-07 — known limitation: ownership-only suggestions are verbose, not topical

`suggest_owners` (Decision 1: a pure heuristic, NOT the `Backend` Protocol) sees
only ownership facts from the `CoverageReport`. Consequences a later slice should
know:

1. **No topic clustering.** Every gap in a fully-unowned file gets its own
   per-file proposed doc id (`pkg/sub/mod.py → pkg-sub-mod`). Files that are
   really *one* topic (e.g. several parser modules) are NOT grouped into one
   proposed doc — the heuristic has no semantic signal to do so. On the dogfood
   repo this yields 524 suggestions, ALL `is_new_doc=True`, one proposed id per
   undocumented module. Correct but verbose.

2. **The intended fix is the deferred LLM-enhanced suggester.** The spec
   explicitly scopes an optional, separate LLM capability out of A-07. When built,
   it should be ADDITIVE: keep `suggest_owners` as the deterministic baseline and
   layer clustering on top, so the offline/mock path stays pure and deterministic
   (K0/K10) and tests never need a network (K4).

3. **Not an EPIC B concern.** EPIC B is region authority
   (`generated|llm|human|llm-seeded`); it does not touch the coverage suggester.
   The only thing EPIC B (and any consumer) should inherit from A-08 is: the
   coverage manifest `.cdmon/coverage.json` is a pure DERIVED, regenerable artifact
   (like the html twins) — never hand-edit it; `cdmon.yaml` stays the single source
   of truth for config, the manifest is a cache refreshed idempotently by
   `cdx coverage --write`.
