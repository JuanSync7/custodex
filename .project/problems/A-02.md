# Problem A-02 — `discover_symbols` aborts the whole scan on one unparseable file

## What

`inventory.discover_symbols(inventory, root)` calls `extract.extract_file(root / path)`
for every python file in the inventory. `extract_file` raises `ExtractionError` (K8) on
an unreadable file or a `SyntaxError`. `discover_symbols` deliberately lets that error
propagate — so **a single syntax-error `.py` anywhere in the tree aborts the entire
symbol inventory**, yielding no `SymbolInventory` at all.

This is the correct, intended behaviour for A-02's target: the dogfood repo and any
known-good first-party codebase, where an unparseable tracked file is a genuine fault we
want surfaced loudly (K8), not silently under-counted.

## Why it's a (future) adoption blocker

EPIC G aims to point cdx at **arbitrary external repositories** to measure
documentation coverage. Such repos routinely contain files that don't parse under our
floor Python:

- vendored / third-party code targeting a different Python version (e.g. `match`
  statements, `print` statements, PEP 695 generics);
- intentionally broken test fixtures (a repo's *own* "this must fail to parse" cases);
- templated / partial `.py` files (Jinja-in-`.py`, cookiecutter sources);
- encoding edge cases beyond `errors="replace"`.

For coverage measurement, one such file should **not** zero out the inventory of the
other (parseable) thousands. "Lossless coverage" wants the bad file *recorded as a gap*,
not used as a hard abort. As-is, A-03+ (ownership/coverage) cannot run against most
real-world external repos without pre-filtering.

## Proposed fix (deferred) — a resilient `--skip-unparseable` mode

Add an opt-in switch to `discover_symbols` (and, later, the `cdx coverage` CLI) that
turns a per-file `ExtractionError` into a recorded gap instead of an abort. Sketch:

```python
class FileSymbols(BaseModel):
    path: str
    language: str
    symbols: tuple[Symbol, ...]
    error: str | None = None          # set when the file could not be parsed

def discover_symbols(
    inventory: Inventory,
    root: Path,
    *,
    skip_unparseable: bool = False,   # default False = current LOUD behaviour (K8)
) -> SymbolInventory: ...
```

- Default `skip_unparseable=False` preserves today's loud K8 contract exactly (no
  existing test changes; first-party / dogfood scans stay strict).
- With `skip_unparseable=True`, a python file whose `extract_file` raises is kept with
  `symbols=()` and `error=<message>` — tracked as a gap (losslessness), the scan
  continues, and the count of skipped files is reportable (`cdx coverage` can warn /
  exit non-zero on a threshold). This keeps determinism (K10): the recorded error string
  is derived from the source, not the clock.

The CLI surface would be `cdx coverage --skip-unparseable` (off by default, so the
strict mode remains the safe default and the resilient mode is an explicit choice).

## Status

Deferred — not needed for A-02..A-06 (first-party dogfood scope). Revisit when EPIC G
(external-repo coverage) lands. No code written for this yet; this note pins the design
so the resilience switch is additive (K9) when it's time.
