## EPIC-CDM result

A standalone, reusable code‚Üídocumentation drift monitor. Config (YAML/JSON) groups
code files ‚ÄĒ to function/line/variable granularity ‚ÄĒ onto audience-tagged logical
documents; the engine extracts an audience-aware surface, detects drift, and a
pluggable LLM backend (mock default / headless claude-code / Anthropic api, chosen
by config) FIXes or INVALIDATEs each drift; every drift+verdict+fix is written to an
append-only review log and emitted to a central sink via a versioned public schema.
Audience requirement met at the strongest level: a user-guide's surface excludes
docstrings/private/local symbols, so a comment change produces NO user-guide drift
while the eng-guide flags it. A Document Layout Standard (v1.0.0) then standardizes
the *shape* of the docs themselves ‚ÄĒ canonical skeleton, unified marker grammar
(helium's spelling as a documented alias), front-matter schema, and the html-twin
pairing rule ‚ÄĒ machine-checked by `cdmon lint` and scaffolded by `cdmon new-doc`,
orthogonal to `check`'s content gate. Offline-testable end to end (no network, no
real LLM); ruff+mypy clean; ‚Č•90% coverage; dogfoods its own config. 224 tests.
