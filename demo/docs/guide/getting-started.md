# Getting started with taskflow

> A friendly, hands-on tour of taskflow for first-time users: build a few
> tasks, wire up their dependencies, and let the engine run them in the right
> order. The "key types" table below is generated by `cdx` from the two types
> you will touch most — it stays in sync with the code automatically.

Taskflow runs small graphs of dependent work. You describe *what* needs to
happen as a handful of tasks, say which task depends on which, and taskflow
figures out a safe order to run them in. There is nothing to configure and no
service to start — it is just a few plain Python objects.

## Your first graph

Create a couple of `Task` objects, drop them into a `TaskGraph`, and hand the
graph to an `Engine`. Each task has an `id`, a human-readable `name`, and the
ids of the tasks it `deps` on:

```python
from taskflow.core import Engine, Task, TaskGraph

graph = TaskGraph()
graph.add(Task(id="fetch", name="Fetch data"))
graph.add(Task(id="report", name="Write report", deps=("fetch",)))

engine = Engine(graph)
completed = engine.run(lambda task: True)  # your runner does the real work
```

The engine walks the graph in dependency order, so `fetch` always runs before
`report`. If a task's runner returns `False`, everything that depends on it is
skipped — you never get a half-finished report built on missing data.
